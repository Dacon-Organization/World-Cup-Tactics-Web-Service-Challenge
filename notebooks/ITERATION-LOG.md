# 분석 반복 사이클 원장 (ITERATION LOG)

> 탐색 → 전처리 → EDA → 가공 → 모델링 → 해석 과정의 **모든 반복(사이클)에 대한 단일
> 대장**입니다. "처음에는 이랬는데, 무슨 문제를 발견해서, 어떻게 수정했는지"가 사라지지
> 않게 하는 것이 목적입니다 ([RESEARCH-LOG](../docs/research/RESEARCH-LOG.md)와 같은 철학).
>
> - 각 노트북 서두의 **개정 이력 셀**은 요약, **이 원장이 정본**입니다.
> - 기록 단위: 사이클(c1, c2, …) = 하류 단계에서 발견한 문제로 상류 단계를 고치는 1회 왕복.
>   최초 작성도 c1로 기록합니다.
> - 규약 상세: [노트북-개정안-2026-07-23](../docs/planning/_changes/노트북-개정안-2026-07-23.md) §4

**최종 갱신**: 2026-07-23

---

## 기록 형식

각 사이클은 아래 6항을 채웁니다. **기각한 대안도 기록**합니다 (같은 실수 반복 방지).

| 항목 | 내용 |
|---|---|
| 발견 | 무엇이 문제였나 — 어느 장의 어떤 산출물·수치에서 |
| 원인 | 왜 발생했나 — 데이터·가정·구현 중 어디의 결함인가 |
| 수정 | 무엇을 어떻게 바꿨나 — 대상 장·셀 |
| 기각한 대안 | 고려했으나 채택하지 않은 수정안과 그 이유 |
| 영향 범위 | 하류 장 중 재실행·재검증이 필요한 곳 |
| 검증 | 수정 후 무엇으로 확인했나 (pandera 게이트·assert·수치 대조) |
