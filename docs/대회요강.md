## 대회 소개

축구 경기를 보면서 누구나 한 번쯤 이런 생각을 합니다. "내가 감독이라면 저 선수 안 쓰는데." "4-3-3이 아니라 3-4-3으로 가야지."

이번 대회에서는 참가자가 바이브 코딩 도구를 활용해 축구 감독이 되어 전술을 직접 구성하고 시각적으로 표현할 수 있는 동적 웹서비스를 설계·구현·배포합니다.

서비스의 구체적인 형태와 콘셉트는 팀이 자유롭게 결정하되, **직접 드래그, 클릭, 배치, 조작하며 감독의 경험을 느낄 수 있어야 합니다.**

## 주제

실제 월드컵 데이터를 활용한 동적 인터랙션 웹서비스 구현

## 설명

본 대회는 실제 월드컵 데이터를 활용하여, 사용자가 축구 감독이 된 것처럼 전술을 직접 구성하고 조작할 수 있는 동적 인터랙션 웹서비스를 구현하는 것을 목표로 합니다.

서비스의 형태와 콘셉트는 자유롭게 기획할 수 있으며, 데이터 활용 방식, UX 구성, 인터랙션 요소, AI 기능 적용 여부 등을 고려하여 독창적인 서비스를 구현합니다.

**예시**

- 드래그앤드롭으로 선수를 배치하고 포메이션을 구성하는 전술 보드
- 상대팀 데이터를 분석하고 AI가 맞춤 전술을 추천해주는 서비스
- 실제 경기 데이터를 기반으로 "내가 감독이었다면" 시나리오를 시뮬레이션하는 서비스
- 경기 스탯을 보며 교체·전술 변경을 지시하는 인터랙티브 서비스

## 평가 안내

### 1차 평가 — 대중 투표

최종 제출한 참가팀을 대상으로 대중 투표를 진행하여 **상위 10팀**을 선정합니다.

| 투표 주체 | 가중치 |
|---|---|
| 제출팀 | **60%** |
| 참가팀 | **20%** |
| 대중 | **20%** |

**동점자 처리 기준**

1. 투표권을 더 많이 사용한 제출팀 우선
2. 산출물 최초 업로드 시간이 빠른 팀 우선

### 2차 평가 — 내부 심사위원 평가

대중 투표 상위 10팀을 대상으로 내부 심사위원 정성 평가를 진행합니다.
**최종 순위는 내부 심사위원 평가 점수로 결정됩니다.**

| 평가 항목 | 주요 평가 요소 | 배점 |
|---|---|---|
| **참신성** | 독창적인 콘셉트, 차별화된 접근 방식, 새로운 경험 제시 | **30점** |
| **감동 경험 설계** | "내가 감독이다"라는 몰입감, 조작의 직관성, UX 흐름 | **25점** |
| **완성도** | 동적 기능이 실제로 작동하며 오류 없이 안정적으로 동작하는가 | **25점** |
| **기획/구현 완성도** | 기획 의도와 실제 구현의 일관성, 핵심 구조 전달력 | **20점** |
| **합계** | | **100점** |

## 개인 또는 팀 참여 규칙

- 개인 또는 팀을 이루어 참여할 수 있습니다.
- **개인 참가**: 팀 신청 없이 자유롭게 참여 가능합니다.
- **팀 참가**: 팀 만들기 또는 팀빌딩을 통해 참여 가능합니다.
- **팀 최대 인원**: 최대 5인
- **중복 참여 제한**: 동일인은 개인 또는 복수 팀에 중복하여 등록할 수 없습니다.
- 팀 빌딩 페이지에서 팀을 구성한 경우, 대회 안내 페이지에서 제출 전 반드시 팀장이 **"팀으로 전환"**을 완료해야 합니다.

> 아래는 원문이 아니라 **원문으로부터 도출한 해석**입니다. 원문과 구분하기 위해 별도 표기합니다.

### 실격·심사제외로 직결되는 조항 (최우선 관리 대상)

| 조항 | 결과 | 우리의 대응 |
|---|---|---|
| 특정 선수·팀·국가 비하 | **심사 제외** | "누가 잘못했는가"가 아닌 "어떤 선택지가 있었는가"로 서술. 카피 한 줄까지 검토 |
| 저작권·라이선스 위반 | **실격 또는 감점** | 선수 사진·엠블럼·유니폼 미사용. 라이선스는 리서치 P2·P7로 검증 |

| 동적 인터랙션 미작동 | 해당 기능 평가 제외 | 배포 후 주요 브라우저·모바일 실동작 검증 필수 |

### 배점이 시사하는 것

- **참신성 30점**이 최대 배점 → 경쟁 서비스 대비 차별점 증명이 가장 중요 (리서치 P1)
- **감동 경험 25점**의 평가 요소가 "몰입감·조작의 직관성·UX 흐름"으로 명시됨 → 기능 명세가 아니라 **경험 설계 원리**로 답해야 함 (리서치 P5)
- **완성도 25점**이 "오류 없이 안정적으로 동작"으로 정의됨 → 화려한 기능보다 **확실히 작동하는 기능**이 유리
- **기획/구현 일관성 20점** → 기획서에 쓴 것과 실제 구현이 어긋나면 감점. **기획서에 과한 약속을 쓰지 않는 것**이 중요

### 1차 통과가 선결 조건

2차 심사 대상은 대중 투표 **상위 10팀**뿐입니다. 아무리 완성도가 높아도 1차에서 탈락하면 심사받지 못합니다.
투표 가중치가 제출팀 60% + 참가팀 20%로 **참가자 비중이 80%** 이므로,
같은 대회에 참가한 개발자들이 봤을 때 **기술적으로 인정할 만한 지점**이 필요합니다.
이것이 ML/DL 깊이를 핵심으로 잡은 근거이기도 합니다.
