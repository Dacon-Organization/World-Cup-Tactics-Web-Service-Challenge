## 개정 이력

### 2026-07-21 — P9 신규 적재 및 P1·P2·P4 개정

**발단**: 사용자가 "Hugging Face 같은 데서 실제 모델이나 데이터를 가져와 쓸 수 있지 않나"를 제기.
다른 팀 프로젝트에서 AI 팀원들이 실제로 그렇게 하는 것을 관찰했다는 근거.
이어서 "가져오더라도 프로젝트에 맞게 커스터마이징하는 것은 자유롭다"는 지도교수 조언을 공유.

**트리거 판정**: T2(대안 교착) + T3(규정 리스크).
"직접 학습 vs 사전학습 모델 파인튜닝"은 사실관계로만 갈리는 노선 선택이며,
파생 모델의 라이선스 상속은 실격과 직결되는 규정 리스크다.

**조치**:
- **P9 신규 작성** — 기존 P3에 욱여넣으면 질문이 10개를 넘어 각 답이 얕아지므로 전용 프롬프트로 분리.
  `PROMPT-TEMPLATE.md`의 "질문은 6개를 넘기지 않는다" 원칙을 따랐다.
- **P2 개정** — 데이터 소스 목록에 Hugging Face Datasets·Kaggle·Zenodo·OpenML 추가.
  산출물 표에 `원출처(재업로드인 경우)` 컬럼 추가.
- **P4 개정** — 런타임 비교에 Transformers.js 추가(내부적으로 ONNX Runtime Web을 쓰므로
  직접 사용 대비 이득·오버헤드를 물음). 모델 호스팅 비교(Vercel public/ vs HF Hub vs 오브젝트 스토리지) 추가.
- **P1 개정** — 인접 영역 조사에 HF Spaces·Streamlit Cloud·Gradio 데모 추가.

**기록해 둘 판단**: 파인튜닝 전제가 성립하면 "모델을 설명하지 못한다"는 우려는 해소된다.
그러나 **라이선스는 파인튜닝으로 세탁되지 않는다.** 이 구분이 P9의 질문 3을
"가장 중요한 질문"으로 명시한 근거다.
