# P1 — 축구 전술보드·라인업 빌더 경쟁 서비스 기능 매트릭스

| 항목 | 내용 |
|---|---|
| **웨이브** | W0 (기반) |
| **트리거** | T5 반박 필요 |
| **블로킹 대상** | 차별화 포지셔닝. 기획서 "경쟁 분석" 절 전체 |
| **실행 순서** | 3번째 |

> **왜 필요한가**: 심사자가 반드시 던질 질문은 **"이거 이미 있는 서비스 아닌가요?"** 입니다.
> 참신성 30점이 여기서 갈립니다. "기존 서비스는 그리기 도구고 우리는 예측한다"는 주장을
> **근거 있는 기능 매트릭스**로 증명해야 합니다. 인상이 아니라 표로.

---

## 사전 조사로 파악한 내용 (검증 대상)

자체 조사에서 확인된 서비스 6종. 모두 드래그앤드롭 배치 + 이미지 내보내기 기능은 있으나,
**배치 결과를 예측·평가하는 기능은 확인되지 않음.**

`buildlineup.com` · `chosen11.com` · `lineup-builder.co.uk` · `footballuser.com` · `tactical-board.com` · `sharemytactics.com`

이 판단이 맞는지, 그리고 **빠뜨린 서비스가 없는지**가 이 리서치의 핵심입니다.

---

## 프롬프트 (복사해서 사용)

```
[역할]
당신은 웹 제품의 경쟁 지형을 분석하는 프로덕트 리서처다.

[배경]
이 Space의 프로젝트 배경을 전제로 한다. 우리는 "배치를 바꾸면 즉시 예측치가 갱신되는"
축구 전술 시뮬레이터를 만들려 한다. 심사에서 참신성이 30점 배점으로 가장 크므로,
기존 서비스와의 차별점을 근거 있게 증명해야 한다.

자체 조사에서 아래 6개 서비스를 확인했고, 모두 "배치하고 이미지로 내보내는 도구"이며
결과를 예측하는 기능은 없는 것으로 파악했다. 이 판단의 검증이 필요하다.
buildlineup.com, chosen11.com, lineup-builder.co.uk, footballuser.com,
tactical-board.com, sharemytactics.com

[질문]
1. 위 6개 서비스 각각의 기능을 확인하고, 우리 판단(예측 기능 없음)이 맞는지 검증하라.
2. 위 6개 외에 축구 전술보드/라인업 빌더/포메이션 에디터 웹서비스를 추가로 찾아라.
   한국어 서비스와 비영어권 서비스도 포함하라. 최소 6개 이상 추가로 제시하라.
3. 발견한 전체 서비스 중, "선수 배치나 전술 설정을 바꾸면 경기 결과·확률·지표를
   예측해서 보여주는" 기능을 가진 것이 하나라도 있는가?
   있다면 그 예측이 어떤 방식(통계 모델/머신러닝/단순 능력치 합산/게임 엔진)인지 밝혀라.
4. 인접 영역까지 넓혀서 확인하라. 다음 각각에서 "전술 조작 → 결과 예측"을 제공하는 사례:
   (a) Football Manager 등 축구 매니저 게임의 웹 버전 또는 웹 도구
   (b) FIFA/EA FC의 스쿼드 빌더류
   (c) 판타지 축구(FPL 등)의 라인업 최적화 도구
   (d) 축구 분석 커뮤니티가 만든 공개 웹 도구 (StatsBomb 데이터 기반 등)
   (e) 스포츠 베팅 예측 사이트의 인터랙티브 시뮬레이터
   (f) Hugging Face Spaces, Streamlit Cloud, Gradio 등에 배포된 축구 분석·예측 데모
       (연구자나 개발자가 만든 프로토타입이 여기 많이 올라온다.
        완성도는 낮아도 아이디어가 겹칠 수 있으므로 반드시 확인하라)
5. 위 조사를 종합할 때, "브라우저에서 로컬 ML 추론으로 전술 변경의 결과를 즉시 예측하는
   웹서비스"는 실제로 비어 있는 영역인가? 가장 근접한 사례는 무엇이며 어떤 점이 다른가?
6. 기존 서비스들이 예측 기능을 넣지 않은 이유를 추정하라.
   기술적 제약인가, 데이터 제약인가, 사용자 수요가 없어서인가?
   이 추정은 우리가 그 기능을 넣었을 때의 리스크를 판단하는 데 쓰인다.

[산출물 형식]
질문 1~4에 대해 다음 컬럼의 표로 통합해서 답하라:
| 서비스명 | URL | 카테고리 | 드래그앤드롭 | 실제 선수DB | 전술 파라미터 조작 | 결과 예측 기능 | 예측 방식 | 데이터 출처 | 무료 여부 | 최종 업데이트 시점 |

"결과 예측 기능" 컬럼은 반드시 다음 중 하나로 표기하라:
없음 / 능력치 단순합산 / 통계모델 / 머신러닝 / 게임엔진 / 미확인

질문 5~6은 서술형으로 답하되, 결론을 먼저 쓰고 근거를 뒤에 배치하라.

[출처 요구]
- 각 서비스는 실제 URL을 제시하라.
- 기능 유무는 서비스 자체의 설명 페이지나 실제 화면 기준으로 판단하라.
  리뷰 기사만으로 판단한 경우 그 사실을 명시하라.
- 접속 불가하거나 서비스가 종료된 경우 그 사실을 기록하라.

[최신성]
- 2025-01 이후에도 운영 중인 서비스를 우선하라.
- 서비스가 중단되었다면 중단 시점을 명시하라.

[배제]
- 마케팅 문구를 기능 설명으로 인용하지 마라. "AI 기반"이라는 표현이 있어도
  실제로 무엇을 하는지 확인되지 않으면 "미확인"으로 표기하라.
- [사실]과 [추정]을 라벨링하라. 질문 6은 대부분 [추정]이 될 것이다.

[언어]
한국어. 서비스명은 원문 표기.
```

## 회수 후 확인 사항

- [ ] 자체 조사 6종에 대한 판단이 **검증 또는 반증**되었는가
- [ ] 추가 서비스가 6개 이상 발견되었는가 (없다면 시장이 정말 좁은 것)
- [ ] "결과 예측 기능" 컬럼이 지정한 6개 값 중 하나로 통일되어 있는가
- [ ] **우리와 정면으로 겹치는 서비스가 발견되었는가** ← 발견되면 콘셉트 재검토 필요

## 이 결과가 결정하는 것

| 결과 | 후속 영향 |
|---|---|
| 예측 기능 서비스 **없음** | 차별화 주장 확정. 기획서에 매트릭스를 그대로 시각자료로 사용 |
| 유사 서비스 **소수 존재** | 그들과의 차이(로컬 추론·한국 시나리오·경우의 수)를 명시적으로 대비 |
| 정면 경쟁 서비스 **존재** | ⚠️ 콘셉트 축을 재조정. 원장에 긴급 항목으로 적재하고 사용자에게 즉시 보고 |
