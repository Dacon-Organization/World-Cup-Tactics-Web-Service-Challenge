# P2 — 축구 데이터 소스 라이선스 및 2026 월드컵 커버리지 감사

| 항목 | 내용 |
|---|---|
| **웨이브** | W0 (기반) |
| **트리거** | T3 규정 리스크 |
| **블로킹 대상** | 데이터 조달 방식 전체. ML 학습 데이터를 무엇으로 할지, 서비스에 어떤 데이터를 실을지 |
| **실행 순서** | **가장 먼저** — 여기 결과에 따라 P3·P4의 전제가 바뀜 |

> **왜 이게 1순위인가**: 대회 규칙상 라이선스 위반은 실격 사유입니다. 그리고 "학습 데이터를 구할 수 있는가"가
> ML 설계 전체의 전제입니다. 여기서 "2026 월드컵 이벤트 데이터는 공개되지 않았다"가 나오면
> 모델 설계를 처음부터 다시 해야 합니다.

---

## 프롬프트 (복사해서 사용)

```
[역할]
당신은 스포츠 데이터 라이선스와 오픈데이터 정책을 검토하는 전문 리서처다.

[배경]
이 Space의 프로젝트 배경을 전제로 한다. 2026 월드컵 데이터를 활용한 웹서비스를
공개 배포하려 하며, 머신러닝 모델 학습에 축구 경기 데이터가 필요하다.
대회 규칙상 라이선스 위반은 실격 사유이고, 주최 측은 실제 정보를 참고해
직접 구성한 더미 데이터 사용을 권장하고 있다.
서비스는 무료로 공개 배포되며 수익을 발생시키지 않는다.

[질문]
1. 아래 데이터 소스 각각에 대해, 무료로 공개된 축구 경기/이벤트 데이터의
   라이선스 조건과 허용 범위를 조사하라:
   StatsBomb Open Data, FBref(Sports Reference), Understat, football-data.co.uk,
   football-data.org, API-Football, Wyscout/Hudl 공개분, Kaggle의 주요 축구 데이터셋,
   openfootball, FIFA 공식 사이트의 공개 통계
2. 각 소스가 "무료 공개 배포되지만 비수익인 웹서비스"에서
   (a) 모델 학습용으로 쓰기, (b) 가공된 파생 지표를 서비스에 표시하기,
   (c) 원본 데이터를 재배포하기 — 각각 허용되는가?
   세 가지를 구분해서 답하라. 이 셋은 라이선스상 전혀 다른 행위다.
3. 2026 FIFA 월드컵(2026년 6~7월 개최, 7/19 종료)의 경기 데이터가
   현재 시점에 어느 수준까지 공개되어 있는가?
   슛 단위 이벤트 데이터 / 경기별 집계 스탯 / 라인업·교체 기록 /
   단순 스코어 — 각 층위별로 공개 여부와 소스를 밝혀라.
4. 선수 이름, 등번호, 포지션, 소속 국가 같은 "사실 정보"를 직접 JSON으로
   구성해 서비스에 싣는 것에 대한 법적 제약이 있는가?
   데이터베이스권(sui generis database right)이 쟁점이 되는 관할과
   그렇지 않은 관할을 구분해서 설명하라.
5. 위 조사 결과를 종합할 때, "라이선스 리스크가 가장 낮으면서
   ML 모델 학습이 가능한" 데이터 조달 경로는 무엇인가?
   1순위와 2순위(대안)를 제시하라.

[산출물 형식]
질문 1~2에 대해 다음 컬럼의 표로 답하라:
| 데이터 소스 | 라이선스 명칭 | 학습 사용 | 파생지표 표시 | 원본 재배포 | 출처 조항 위치 | 라이선스 원문 URL |

질문 3에 대해 다음 컬럼의 표로 답하라:
| 데이터 층위 | 2026 WC 공개 여부 | 소스 | 접근 방법 | 확인 시점 |

질문 4~5는 서술형으로 답하되, 결론을 먼저 쓰고 근거를 뒤에 배치하라.

[출처 요구]
- 라이선스는 반드시 원문 조항 URL을 제시하라. 요약 블로그를 1차 출처로 쓰지 마라.
- 조항 내용을 인용할 때는 어느 섹션인지 명시하라.
- 확인 불가한 항목은 "미확인"으로 남겨라.

[최신성]
- 2025-01 이후 정보를 우선하라. 라이선스는 개정될 수 있으므로 확인 시점을 명시하라.
- 2026 월드컵 데이터 공개 현황은 대회가 2026-07-19에 종료된 직후이므로,
  가능한 한 최근 시점의 정보를 확인하라.

[배제]
- "일반적으로 괜찮을 것이다" 같은 서술을 배제하라. 조항 근거가 없으면 "미확인"이다.
- [사실]과 [추정]을 라벨링하라.
- 법률 자문이 아님을 전제하되, 조항 원문에 근거한 판단은 명확히 제시하라.

[언어]
한국어. 라이선스 명칭과 법률 용어는 원문 병기.
```

## 회수 후 확인 사항

- [ ] 각 소스의 **학습 / 표시 / 재배포** 세 행위가 구분되어 답변되었는가
- [ ] 라이선스 원문 URL이 실제로 접근 가능한가 (블로그 요약이 아닌지)
- [ ] 2026 월드컵 이벤트 데이터 공개 여부에 대한 명확한 답이 있는가
- [ ] 1순위 조달 경로가 대회 규칙(더미 데이터 권장)과 충돌하지 않는가

## 이 결과가 바꿀 수 있는 것

| 결과 | 후속 영향 |
|---|---|
| 2026 WC 이벤트 데이터 **공개됨** | 실데이터 기반 xG 모델 학습 가능 → P3 설계 그대로 진행 |
| 2026 WC 이벤트 데이터 **미공개** | 과거 대회 데이터로 학습 + 2026은 큐레이션 JSON으로 구성 → P3에 전이학습 관점 추가 필요 |
| 주요 소스가 **재배포 금지** | 모델 가중치만 배포하고 원본 데이터는 레포에 넣지 않는 구조로 전환 |
