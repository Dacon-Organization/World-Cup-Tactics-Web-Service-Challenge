# P3 — 축구 결과 예측·xG 모델링의 검증된 기법과 한계

| 항목 | 내용 |
|---|---|
| **웨이브** | W0 (기반) |
| **트리거** | T2 대안 교착 |
| **블로킹 대상** | M1(xG)·M2(결과예측)·M3(포메이션 상성) 모델 기법 선정 |
| **실행 순서** | 4번째 |

> **왜 필요한가**: 사용자가 이 프로젝트에서 가장 중요하게 본 것이 **데이터분석과 ML/DL의 깊이**입니다.
> 여기서 "그냥 능력치 합산"으로 가면 기존 서비스와 다를 게 없어집니다.
> 반대로 근거 없이 딥러닝을 갖다 붙이면 **과적합된 장난감**이 됩니다.
> 검증된 기법과 그 한계를 알아야 "왜 이 모델을 골랐는가"를 기획서에서 방어할 수 있습니다.

---

## 프롬프트 (복사해서 사용)

```
[역할]
당신은 스포츠 애널리틱스와 통계 모델링을 다루는 연구 리서처다.

[배경]
이 Space의 프로젝트 배경을 전제로 한다. 우리는 브라우저에서 실시간으로 동작하는
축구 전술 시뮬레이터를 만들며, 다음 모델들이 필요하다:
- M1: 슛 상황 → 득점 확률 (xG 모델)
- M2: 팀 강도 → 스코어라인 확률 분포 (경기 결과 예측)
- M3: 전술 파라미터 조합(라인 높이·압박 강도·템포·폭) → 기대 득점 차
- M4: 선수 특성 → 포지션 적합도
제약: 모델은 ONNX로 내보내 브라우저에서 추론하므로 크기와 연산량이 작아야 한다.
학습 데이터는 제한적일 수 있다(P2 리서치 결과에 의존).

[질문]
1. xG(기대득점) 모델의 표준적 구현을 조사하라.
   일반적으로 쓰이는 입력 피처는 무엇이며, 어떤 알고리즘이 쓰이는가?
   공개된 xG 모델 구현체 중 참고할 만한 것과 그 성능 지표(AUC, Brier score 등)를 제시하라.
2. 축구 경기 결과 예측의 검증된 통계 모델을 조사하라.
   Dixon-Coles 모델, 포아송 회귀, Elo/Glicko 계열 레이팅,
   베이지안 계층 모델, 머신러닝 접근 각각의 특징과 실측 정확도를 비교하라.
   특히 "국가대표팀 경기"와 "단기 토너먼트"에 적용할 때의 특수성을 밝혀라.
3. 포메이션이나 전술 스타일이 경기 결과에 미치는 영향을 정량화한 연구가 있는가?
   포메이션 상성(예: 4-3-3 vs 3-5-2)을 데이터로 검증한 사례,
   압박 강도(PPDA 등)나 라인 높이 같은 전술 지표가 결과와 갖는 상관관계 연구를 조사하라.
   이런 연구가 부족하다면 그 사실 자체를 명확히 밝혀라.
4. 선수를 익숙하지 않은 포지션에 배치했을 때의 성능 저하를 모델링한 사례가 있는가?
   포지션 적합도나 선수 역할 임베딩을 다룬 연구·공개 구현을 조사하라.
5. 위 모델들을 "학습 데이터가 수백~수천 경기 수준"으로 구축할 때의 현실적 한계는 무엇인가?
   과적합 위험, 신뢰구간의 폭, 국가대표 경기의 표본 부족 문제를 구체적으로 지적하라.
   그리고 이런 한계를 사용자에게 정직하게 표시하는 방법(불확실성 시각화)의 사례를 제시하라.
6. 몬테카를로 시뮬레이션으로 토너먼트 진출 확률을 계산하는 표준적 방법을 설명하라.
   FiveThirtyEight, Opta, The Athletic 등이 사용한 방법론 중 공개된 것을 참고하라.
   몇 회 반복이면 확률이 안정적으로 수렴하는가?

[산출물 형식]
질문 1~2에 대해 다음 컬럼의 표로 답하라:
| 모델/기법 | 입력 피처 | 알고리즘 | 보고된 성능 | 데이터 요구량 | 계산 비용 | 공개 구현체 | 출처 URL |

질문 3~4는 다음 컬럼의 표로 답하라:
| 연구/구현 | 다룬 문제 | 방법 | 결론 | 한계 | 출처 URL |

질문 5~6은 서술형으로 답하되, 우리 제약(작은 모델, 제한된 데이터, 브라우저 추론)에
비추어 무엇을 권고하는지 명확히 밝혀라.

[출처 요구]
- 학술 논문은 저자·연도·저널/학회를 명시하고 DOI 또는 접근 URL을 제시하라.
- 블로그 포스트를 인용할 경우 그것이 1차 연구인지 요약인지 구분하라.
- 성능 수치는 반드시 평가 데이터셋과 함께 제시하라. 데이터셋이 다르면 비교 불가다.

[최신성]
- 2020년 이후 연구를 우선하되, Dixon-Coles(1997)처럼 표준이 된 고전은 포함하라.
- 고전 문헌은 발표 연도를 명시하라.

[배제]
- 베팅 예측 상품의 마케팅 주장(적중률 몇 % 등)을 근거로 쓰지 마라.
- 재현 가능한 방법론이 공개되지 않은 성능 주장은 [추정]으로 라벨링하라.
- 우리 제약에 맞지 않는 대규모 모델(트래킹 데이터 필요, GPU 추론 필요 등)은
  그 사실을 명시하고 "적용 불가" 판정을 내려라.

[언어]
한국어. 모델명·통계 용어는 원문 병기.
```

## 회수 후 확인 사항

- [ ] 각 모델의 **데이터 요구량**이 우리가 확보 가능한 규모와 맞는가 (P2 결과와 대조)
- [ ] 브라우저 추론이 가능한 **경량 모델**로 좁혀졌는가
- [ ] 포메이션 상성 연구가 **실제로 존재하는가** — 없다면 M3를 휴리스틱으로 낮춰야 함
- [ ] 불확실성 표시 방법에 대한 구체적 사례가 있는가

## 이 결과가 결정하는 것

- M1~M4 각각의 **알고리즘 확정**
- **모델을 포기할 것인지**의 판단 — 예를 들어 포메이션 상성 연구가 빈약하면
  M3는 ML이 아니라 명시적 룰로 구현하고, 기획서에 그 이유를 정직하게 밝히는 편이 낫습니다
- 기획서 "ML/DL 아키텍처" 절의 **기법 선정 근거**

> ⚠️ **주의**: 이 리서치의 결과가 "축구 결과 예측은 본질적으로 불확실성이 크다"로 나올 가능성이 높습니다.
> 그건 나쁜 소식이 아닙니다. **불확실성을 정직하게 보여주는 것 자체를 제품의 특징으로** 만들면
> 오히려 신뢰도가 올라갑니다. 확률을 단정적 예측처럼 표시하는 것이 진짜 리스크입니다.
