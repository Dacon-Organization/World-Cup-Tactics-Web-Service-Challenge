# P5 — 드래그앤드롭 전술보드의 UX·접근성·몰입 설계

| 항목 | 내용 |
|---|---|
| **웨이브** | W0 (기반) |
| **트리거** | T1 근거 공백 |
| **블로킹 대상** | 인터랙션 명세, 기획서 "감독 경험 설계 의도" 절 |
| **실행 순서** | 6번째 |

> **왜 필요한가**: 심사 배점의 **감동 경험 설계 25점**이 걸린 항목입니다.
> 평가 요소가 명시적으로 **"내가 감독이다라는 몰입감, 조작의 직관성, UX 흐름"** 입니다.
> "드래그앤드롭 되게 만든다"는 기능 명세지 경험 설계가 아닙니다.
> **왜 그 조작이 감독이 된 기분을 만드는가**를 설계 원리로 설명할 수 있어야 합니다.

---

## 프롬프트 (복사해서 사용)

```
[역할]
당신은 인터랙션 디자인과 게임 UX를 다루는 디자인 리서처다.

[배경]
이 Space의 프로젝트 배경을 전제로 한다. 우리는 사용자가 "축구 감독이 된 것 같은"
몰입감을 느끼는 전술보드를 만들려 한다. 심사 배점에서 "감동 경험 설계"가 25점이며,
평가 요소는 "'내가 감독이다'라는 몰입감, 조작의 직관성, UX 흐름"이다.

핵심 인터랙션:
- 선수 카드를 필드 위에 드래그앤드롭으로 배치
- 전술 파라미터(라인 높이·압박 강도·템포·폭)를 슬라이더로 조정
- 조작할 때마다 예측치(xG·승패 확률·진출 확률)가 즉시 갱신됨
- 조별리그 결과 조합을 조작해 진출 확률 확인

[질문]
1. 웹에서 드래그앤드롭 인터랙션을 구현할 때의 UX 모범 사례를 조사하라.
   드래그 중 시각적 피드백, 드롭 가능 영역 표시, 스냅 동작, 취소/되돌리기,
   잘못된 드롭에 대한 처리 방식을 구체적으로 다뤄라.
2. 드래그앤드롭의 접근성 문제와 해결책을 조사하라.
   키보드만으로 조작하는 방법, 스크린리더 대응(ARIA live region 등),
   WCAG 관련 기준을 포함하라.
   또한 모바일 터치 환경에서 드래그앤드롭이 겪는 고유한 문제
   (스크롤과의 충돌, 터치 타겟 크기, 손가락에 가려지는 영역)와 해결 패턴을 조사하라.
3. "역할 몰입(role immersion)"을 만드는 인터페이스 설계 원리를 조사하라.
   사용자가 특정 역할(감독, 지휘관, 관제사 등)이 된 것처럼 느끼게 하는
   UI 요소·정보 배치·용어 선택·피드백 방식에 대한 연구나 사례를 찾아라.
   축구 매니저 게임(Football Manager 등)이 이를 어떻게 구현했는지 포함하라.
4. 조작에 대한 즉각적 피드백이 몰입에 미치는 영향을 다룬 연구를 조사하라.
   어느 정도의 지연(latency)부터 사용자가 "반응이 느리다"고 인지하는가?
   조작과 결과 표시 사이의 시간에 대한 구체적 임계값 수치를 제시하라.
5. 확률과 불확실성을 사용자에게 전달하는 시각화 방법을 조사하라.
   단일 수치로 표시할 때의 오해 문제, 확률을 직관적으로 전달하는 대안
   (아이콘 배열, 분포 시각화, hypothetical outcome plots 등),
   그리고 스포츠 예측 서비스가 실제로 쓰는 방식을 포함하라.
6. 첫 방문자가 복잡한 도구를 이해하게 만드는 온보딩 설계를 조사하라.
   튜토리얼 없이도 조작 가능하게 하는 어포던스 설계,
   점진적 공개(progressive disclosure), 기본값(default)의 중요성을 다뤄라.
   특히 "심사위원이 짧은 시간 안에 핵심 기능을 발견해야 하는" 상황에
   적용할 수 있는 원칙을 제시하라.

[산출물 형식]
질문 1~2에 대해 다음 컬럼의 표로 답하라:
| 문제/상황 | 권장 패턴 | 이유 | 안티패턴 | 관련 기준(WCAG 등) | 출처 URL |

질문 3~6은 서술형으로 답하되, 각 항목마다
"우리 서비스에 적용한다면 구체적으로 무엇을 해야 하는가"를 마지막에 덧붙여라.
질문 4와 5는 반드시 구체적 수치나 임계값을 포함하라.

[출처 요구]
- Nielsen Norman Group, WCAG, 디자인 시스템 공식 문서 등을 1차 출처로 우선하라.
- 학술 연구를 인용할 경우 저자·연도를 명시하라.
- 게임 사례는 실제 화면이나 공식 설명을 근거로 하라.

[최신성]
- 2022년 이후 자료를 우선하되, 인지심리학 기반의 고전 연구
  (반응 시간 임계값 등)는 연도와 함께 포함하라.

[배제]
- "직관적으로 만들어야 한다" 같은 동어반복적 조언을 배제하라.
  무엇을 어떻게 해야 직관적이 되는지 구체적 패턴으로 답하라.
- 특정 UI 라이브러리의 홍보성 문서를 모범 사례로 인용하지 마라.
- [사실]과 [추정]을 라벨링하라.

[언어]
한국어. 디자인 용어·패턴명은 원문 병기.
```

## 회수 후 확인 사항

- [ ] 모바일 터치 드래그의 **구체적 해결 패턴**이 나왔는가 (심사자 모바일 접속 대비)
- [ ] 반응 지연 **임계값 수치**가 나왔는가 → P4의 성능 예산과 대조 필요
- [ ] 확률 시각화 방법이 구체적 사례와 함께 나왔는가
- [ ] "역할 몰입" 원리가 **적용 가능한 형태**로 정리되었는가

## 이 결과가 결정하는 것

- 기획서 **"감독 경험 설계 의도"** 절의 논리 구조 (25점 배점 항목)
- 인터랙션 명세의 상태 전이 설계
- 접근성 대응 범위
- **확률 표시 방식** — P3의 "불확실성을 정직하게 보여준다"와 직결

## P4와의 교차 검증 지점

질문 4에서 나오는 **"사용자가 지연을 인지하는 임계값"** 과
P4에서 나오는 **"실제 추론 지연"** 을 반드시 대조해야 합니다.

- 추론 지연 < 인지 임계값 → 실시간 갱신 가능
- 추론 지연 > 인지 임계값 → **디바운싱 + 로딩 상태 표시**로 설계 변경 필요

이 대조 결과는 원장에 기록합니다.
