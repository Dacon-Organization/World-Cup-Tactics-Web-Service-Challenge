|---|---|

| **트리거** | T3 규정 리스크 |

| **선행 리서치** | [P2](../findings/P2-데이터라이선스.md) · [P9](../findings/P9-사전학습모델활용.md) 후속 |

> "미국·한국의 데이터베이스권 1차 법령"을 확인하지 못한 채 [추정]/[미확인]으로 남겼습니다.
> 2순위 조달 경로(Kaggle/HF CC0·MIT·Apache 데이터셋)를 실제로 채택할지 결정하려면 이 공백을 메워야 합니다.
> 이 질문은 설계와 무관하게 지금 발주 가능합니다.

## 프롬프트 (복사해서 사용)

```
[역할]
당신은 스포츠 데이터 라이선스와 지식재산권(저작권·데이터베이스권) 법령을 검토하는 전문 리서처다.

[배경]
이 Space의 프로젝트 배경을 전제로 한다. 2026 월드컵 데이터를 활용한 웹서비스를 무료로 공개 배포하며,
ML 모델 학습에 축구 데이터가 필요하다. 대회 규칙상 라이선스 위반은 실격 사유다.
이전 리서치(P2·P9)에서 다음을 확인하지 못한 채 남겨두었고, 이번에 그 공백을 메우려 한다:
- Hugging Face/Kaggle에 올라온 재업로드 데이터셋의 카드 표기 라이선스가 원출처의 공식 라이선스와
  문구 단위로 일치하는지 대조하지 못했다.
- 미국·한국·EU에서 "선수 실명·포지션·스코어 같은 사실 데이터의 집합"을 직접 타이핑해 재구성하는 행위가
  데이터베이스권을 침해하는지, 1차 법령 조문으로 확인하지 못했다.

[질문]
1. 아래 재업로드 데이터셋 각각에 대해, 허브 카드의 license 필드에 표기된 라이선스와
   원출처(StatsBomb, FBref/Sports Reference, Kaggle 원본 등)의 공식 라이선스 전문을
   문구 단위로 대조하라. 불일치가 있으면 어느 쪽이 더 보수적(제한적)인지 밝혀라:
   - ClementeH/statsbomb-open-data-shots, FelixCS3/statsbomb-open-data-shots (HF)
   - roirani80/fbref-xg-analysis-2024-2025 (HF)
   - Mominullptr/fifa-world-cup-2026-dataset (HF/Kaggle)
   - kulkarniparth09/fifa-world-cup-complete-dataset-19302026 (Kaggle)
   - joshfjelstul/world-cup-database (Kaggle)
2. P2에서 "미확인"으로 남은 아래 소스의 이용약관 원문을 확인하라(존재 여부 포함):
   Understat 자체 ToS, Wyscout/Hudl "공개분"의 존재 여부와 라이선스, OpenML의 축구 데이터셋과 그 라이선스.
3. StatsBomb Open Data의 LICENSE.pdf 전문 조항을 확인하라(P2에서 GitHub 뷰어 렌더 실패로 미확인):
   상업적 이용 가부, 원본 재배포 조건, 출처표기·로고 의무의 정확한 문구.
4. 다음 3개 관할에서 "선수 실명·등번호·포지션·소속국·스코어 같은 사실 데이터의 집합"을
   여러 독립 출처를 참고해 직접 타이핑하여 JSON으로 재구성하는 행위가
   데이터베이스권 또는 저작권을 침해하는지, 조문 원문을 인용해 답하라:
   - 미국: 17 U.S.C.(사실 데이터·데이터 집합의 보호 부재 여부)
   - 한국: 저작권법 제93조(데이터베이스제작자의 권리)
   - EU: Database Directive 96/9/EC 제7조(sui generis 권리)
5. 위를 종합해, 2순위 경로(Kaggle/HF의 CC0·MIT·Apache 데이터셋)를 채택해도 안전한 조건과,
   채택을 금지해야 할 위험 신호(red flag)를 목록으로 제시하라.

[산출물 형식]
질문 1~3은 다음 컬럼의 표로 답하라:
| 데이터셋/소스 | 카드 표기 라이선스 | 원출처 공식 라이선스 | 문구 일치 여부 | 보수적 채택안 | 원문 URL |

질문 4는 다음 컬럼의 표로 답하라:
| 관할 | 법령/조문 | 사실 데이터 집합의 보호 여부 | 직접 재구성 JSON의 위험도 | 조문 원문 URL |

질문 5는 서술형(결론 먼저, 근거 뒤).

[출처 요구]
- 라이선스·법령은 반드시 원문 조항 URL을 제시하라. 요약 블로그를 1차 출처로 쓰지 마라.
- 법령은 조·항·호 번호를 명시하라. 판례를 인용할 경우 사건번호와 판결 원문을 우선하라.
- 확인 불가한 항목은 "미확인"으로 남겨라. 추측으로 채우지 마라.

[최신성]
- 2025-01 이후 정보를 우선하고, 라이선스·법령은 확인 시점을 병기하라.

[배제]
- "일반적으로 괜찮을 것이다" 서술 배제. 조항 근거가 없으면 "미확인"이다.
- 법률 자문이 아님을 전제하되, 조문 원문에 근거한 판단은 명확히 제시하라.
- [사실]과 [추정]을 라벨링하라.
- 이 Space의 이전 리서치(P2·P9)와 모순되는 내용이 있으면 명시적으로 지적하라.

[언어]
한국어. 라이선스 명칭·법률 용어는 원문 병기.
```

## 회수 후 확인 사항

- [ ] 각 재업로드 데이터셋의 카드 표기와 원 라이선스 **문구 일치 여부**가 판정되었는가
- [ ] StatsBomb LICENSE.pdf의 상업/재배포/출처표기 조건이 문구로 확인되었는가
- [ ] 미국·한국·EU 각 관할의 **조문 원문**이 인용되었는가 (요약이 아닌지)
- [ ] 2순위 채택 조건과 red flag 목록이 실무적으로 판단 가능한가

## 이 결과가 바꿀 수 있는 것

| 결과 | 후속 영향 |
|---|---|
