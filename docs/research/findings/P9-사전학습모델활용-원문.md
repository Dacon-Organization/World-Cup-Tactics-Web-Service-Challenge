### 라이선스 상속 원칙 (핵심 오해 교정)

"파인튜닝하면 라이선스가 리셋된다"는 통념은 사실과 다르다. Apache-2.0, MIT 등 permissive 라이선스는 파생 모델에도 원 라이선스 조항(저작권고지·변경사항 표시 등)이 승계되지만 재라이선싱 제약은 약하다. 반면 CC-BY-NC-4.0이나 TabPFN-2.5 커스텀 라이선스처럼 조건이 부착된 라이선스는, 파인튜닝된 파생물에도 동일한 비상업 조건이 그대로 승계된다는 것이 원칙이다. Hugging Face Hub 커뮤니티에서도 "Llama/Alpaca 계열 모델이 비상업 데이터로 학습되었음에도 파생 모델의 라이선스가 실제보다 더 개방적으로 표기되는 사례"가 지적된 바 있다. 즉 학습에 사용한 데이터나 베이스 모델 중 하나라도 비상업·CC-BY-NC 조건이면, 최종 파생 모델도 원칙적으로 그 조건을 벗어날 수 없다.[^27]

## 질문 1 결과 요약: 축구 특화 모델·데이터셋 생태계 품질

Hugging Face Hub에 존재하는 축구/스포츠 특화 모델은 대부분 개인 취미 프로젝트 수준이며 독립적 벤치마크나 논문 검증이 없다. AmjadKha/FootballerModel은 정확도 0.49로 무작위 추측(3-class 기준 약 0.33)보다 나은 수준일 뿐 실용성이 낮다. sergiopesch/wc2026-match-predictor는 499경기라는 소량 데이터로 5-fold CV 정확도 0.545(베이스라인 0.455 대비)를 기록했으나, 모델 카드 스스로 "form·injury·전술을 반영하지 않는 3개의 조악한 피처"라고 명시해 한계를 인정한다. 학계 수준 연구는 SoccerChat(비디오-언어 멀티모달, SimulaMet 개발)과 Forecasting Events in Soccer Matches Through Language(WyScout 데이터 기반 LEM) 정도이나 두 모델 모두 브라우저 로컬 추론에는 부적합한 대형 모델이다. 축구 데이터셋(StatsBomb·FBref 재업로드본)은 존재하지만 라이선스 필드가 비어 있거나 불명확한 경우가 많아, "쓸 만한 품질"과 "라이선스가 검증된 상태"를 모두 만족하는 항목은 거의 없다는 점을 명확히 밝힌다.[^1][^28][^6][^3][^8][^5][^4]

## 질문 2 결과: 소형 표 데이터 파운데이션 모델의 실효성

TabPFN v2/v2.5는 Nature(2025) 논문에서 최대 1만 샘플 규모까지 기존 방법을 큰 격차로 능가한다고 보고되었다. 독립적 3자 벤치마크(Kaggle 데이터셋 기준)에서도 TabPFN v2가 Titanic(정확도 0.789 vs XGBoost 0.736, CatBoost 0.775), House Prices(RMSLE 0.114 vs XGBoost 0.156), Rainfall(AUC 0.866 vs XGBoost 0.846)에서 GBDT를 앞섰다. 다만 동일 벤치마크에서 시계열 성격의 "Sticker Sales" 과제는 XGBoost가 더 우수했고, 1만 행을 초과하면 메모리 문제로 실행이 어렵다는 한계가 확인되었다. 이는 "수백~수천 경기" 규모의 이 프로젝트 데이터와 정확히 일치하는 조건이므로, TabPFN 계열이 GBDT 대비 우위를 가질 가능성은 [추정]이 아니라 독립 벤치마크로 뒷받침된 사실에 가깝다. 단, 이 벤치마크는 축구 도메인이 아닌 범용 Kaggle 데이터셋 기준이므로 축구 경기 예측에 그대로 일반화된다고 단정할 수는 없다(추정).[^29][^30][^31]

시계열 파운데이션 모델(Chronos-Bolt, TimesFM, Moirai)은 제로샷 시계열 예측(경기당 통계 흐름, 팀 폼 추이 등)에는 적합할 수 있으나, 이들 모델 카드의 성능 주장은 27개 범용 시계열 데이터셋 벤치마크에 기반한 것으로 축구 도메인 검증은 없다. 축구 전술 시뮬레이터의 핵심 과제(경기 결과·xG 예측)는 시계열이 아니라 정형(tabular) 분류/회귀 문제에 더 가깝기 때문에, 시계열 파운데이션 모델의 적합성은 제한적이라고 판단한다[추정].[^14]

## 질문 4: Hugging Face 데이터셋 라이선스 표기 신뢰성 검증 절차

Hugging Face Hub 커뮤니티 포럼에는 라이선스 오표기 사례가 명확히 기록되어 있다. Alpaca 계열 데이터셋·모델이 원본이 비상업 조건임에도 더 개방적인 라이선스로 표기된 사례가 지적되었고, 저작권자 동의 없이 콘텐츠가 재라이선싱되어 업로드된 사례도 보고되었다. 즉 데이터셋 카드의 `license` 필드는 업로더의 자기 신고이며 HF가 검증하지 않는다는 점을 전제해야 한다.[^32][^27]

**검증 체크리스트**
- 데이터셋 카드에 기재된 원출처(StatsBomb, FBref, WyScout 등)를 명시적으로 확인한다. 원출처가 없으면 그 자체로 위험 신호로 간주한다.
- 원출처의 공식 라이선스 페이지·리포지토리(예: github.com/statsbomb/open-data/LICENSE.pdf)를 직접 열어 조건 전문을 대조한다.[^7][^25]
- HF 카드의 `license` 필드와 원출처 조건이 문구 단위로 일치하는지 비교한다. 불일치 시 더 제한적인(보수적인) 쪽을 채택한다.
- 데이터셋의 Discussions/Community 탭에서 라이선스 이의제기나 삭제 요청 이력을 검색한다.[^32]
- 업로더가 원 제작자 본인인지, 제3자 재업로드인지 확인한다(리포지토리명·소속 계정 확인).
- FBref의 경우 Sports Reference LLC 이용약관을 별도로 확인해야 한다(미확인 — 이번 리서치에서 원문 대조 미완료).
- 최종적으로 "재현 가능한 원문 URL"을 확보하지 못하면 그 데이터셋은 채택하지 않는다.

ONNX Runtime Web의 공식 문서는 protobuf 파일 최대 2GB, WASM 메모리 최대 4GB라는 하드 리밋을 명시하며, 이 한도 내에서는 실행 가능하지만 "모델이 너무 크면 저사양 기기에서 비효율적"이라는 일반적 경고를 제공한다. 그러나 이번 리서치에서 TabPFN이나 Chronos-Bolt/TimesFM/Moirai를 실제로 ONNX로 변환해 브라우저에서 구동한 사례나 정량적 파일 크기·지연시간 수치는 확인하지 못했다. 양자화·증류 시 성능 손실 폭에 대한 수치도 이번 검색에서 확보하지 못했다(미확인). TabPFN 계열은 특히 "in-context learning" 방식으로 추론 시 학습 데이터셋 전체를 프롬프트처럼 입력해야 하는 구조이므로, 고정 그래프 기반 ONNX 변환 및 오프라인 사전 컴파일에 근본적으로 적합하지 않을 가능성이 있다(추정 — 검증 필요).[^34][^33]

## 질문 6: 종합 권고

**권고: 노선 (A) 직접 학습을 채택하고, 필요시 TabPFN을 오프라인 벤치마크·피처 엔지니어링 참고용으로만 활용하는 절충안을 취한다.** LLM API 없이 100% 브라우저 로컬 추론이라는 대회 제약과, TabPFN·Chronos·TimesFM·Moirai 등 파운데이션 모델의 라이선스·변환 리스크를 종합하면 직접 학습이 명확히 더 안전하다.

- **모델 성능**: TabPFN v2는 수백~수천 행 규모에서 GBDT를 능가한다는 독립 벤치마크가 존재하지만, 이는 범용 Kaggle 데이터 기준이며 in-context learning 구조상 브라우저 오프라인 추론에 적합한지 확인되지 않았다. 반면 소형 GBDT(XGBoost/LightGBM)나 경량 신경망은 수백~수천 경기 규모에서도 충분히 안정적으로 학습되며, sergiopesch 사례처럼 499경기로도 베이스라인 대비 개선을 보였다.[^3][^30][^29]
- **라이선스 리스크**: TabPFN-2.5는 상업/프로덕션 목적 사용을 전면 금지하는 비상업 라이선스이며, TabPFN v2는 Apache-2.0 기반이나 추가 출처표기 의무가 있는 커스텀 조항이라 정확한 허용 범위가 원문 대조 없이는 확정되지 않는다. Moirai 2.0/1.1은 CC-BY-NC-4.0으로 비상업 조건이 붙는다. 대회 규정상 "라이선스 위반 시 실격"이라는 조항의 무게를 고려하면, 조건이 복잡한 파운데이션 모델보다 처음부터 직접 학습한 모델(라이선스 전체를 스스로 통제 가능)이 리스크가 현저히 낮다.[^10][^23][^24][^13]
- **브라우저 배포 가능성**: 직접 학습한 소형 GBDT나 로지스틱 회귀는 ONNX 변환 후 수백KB~수MB 수준으로 WASM 환경에서 확실히 구동 가능하다. 반면 TabPFN·Chronos-base·TimesFM 등은 변환 사례 자체가 검증되지 않았고, TabPFN은 구조적으로 표준 ONNX 고정 그래프 변환에 부적합할 가능성이 있다(추정).[^33]
- **기획서에서 설명 가능한가**: 심사 배점상 "기획/구현 완성도 20점"은 기획 의도와 구현의 일관성을 요구한다. "축구 데이터 수백~수천 경기로 직접 학습한 경량 모델을 ONNX로 내보내 브라우저에서 추론한다"는 서술은 명확하고 검증 가능하다. 반면 "Nature 논문에 실린 파운데이션 모델을 파인튜닝했다"는 서술은 인상적이지만, 라이선스 조건과 실제 변환 가능성을 자신 있게 설명하기 어려워 심사자의 반박 질문(T5)에 취약하다.

절충안으로는 Chronos-Bolt-tiny(9M, Apache-2.0)나 Moirai-1.0-R-small(13.8M, Apache-2.0)처럼 완전 개방 라이선스이고 파라미터 규모가 작은 시계열 모델을 팀 폼 추이 같은 보조 피처 추출에 실험적으로 활용하고, 핵심 예측(경기 결과·xG)은 직접 학습한 GBDT/경량 신경망으로 처리하는 하이브리드 구조가 가능하다. 단, 이 절충안도 실제 ONNX 변환 검증(질문 5의 미확인 항목)을 거친 뒤에만 채택해야 한다.[^20][^14]

***

## 이 답변의 한계

TabPFN v2/v2.5 및 Chronos/TimesFM/Moirai를 실제로 ONNX로 변환해 브라우저에서 구동한 사례, 파일 크기, 지연시간, 양자화 손실 수치는 확인하지 못했으며 질문 5의 다수 항목이 미확인으로 남아 있다. StatsBomb·FBref 재업로드 데이터셋의 HF 카드 라이선스 필드와 원 라이선스 전문의 문구 단위 대조도 완료하지 못했으므로, 실제 사용 전 원문 재확인이 필요하다. TabPFN 계열 라이선스 원문(LICENSE.txt) 전체 조항 역시 요약 수준으로만 확인했으며, "파인튜닝된 파생물"에 대한 구체적 허용 범위는 Prior Labs에 직접 문의하는 것이 안전하다.
