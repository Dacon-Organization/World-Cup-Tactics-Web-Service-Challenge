gh api 실측 (2026-07-23):

| `fbdesignpro/sweetviz` | 3,118★ | MIT | 2026-04-11 | 보류 (기능 중복) |

**⚠️ 신선도 주의 — 개명 사실 (README 원문 확인)**: `pandas-profiling` → `ydata-profiling` → **`fg-data-profiling`** (2026-04 개명). 공지 원문: "This package has been renamed to `fg-data-profiling`. … the old package will no longer receive updates or bug fixes." 설치는 `pip install fg-data-profiling`, import는 `data_profiling`. **웹 튜토리얼 대부분이 옛 이름이므로 혼동 주의.** PyPI 실측: 최신 4.18.4(2026-04-22), `pandas >1.5,<3.0` (우리 스택 정합), Python ≥3.10 <3.14.

**채택 조건 3항 (커스터마이징 방침)**:
1. **마스킹 이후에만 실행** — 프로파일 리포트는 컬럼 값 샘플을 노출하므로 01장 마스킹 셀(NB-R1) **이후** 데이터프레임에만 적용. 실명 유출 경로 차단.
2. **HTML 통짜 임베드 금지** — `minimal=True` 모드 + `to_json()`(또는 `describe` 객체)에서 필요한 수치만 추출해 **커스텀 요약 표**로 노트북에 표시. 수 MB HTML을 출력 셀에 넣으면 커밋 노트북이 열람 불가가 됨.
3. **도구는 보조, 서사는 사람** — 프로파일 수치는 "탐색의 출발점 표"로 쓰고, 각 이상 신호(결측·왜도·중복)에 대한 해석은 마크다운 서사로 직접 작성 (노트북_서사 §1 원칙 유지).

missingno는 결측 시각화 3종(matrix·bar·heatmap)만 쓰는 초경량 채택 — 00장 탐색 절 전용.

### Q5. 전문가 워크플로 규약 — cookiecutter-data-science 부분 채택 + 스킬화 제안

`drivendataorg/cookiecutter-data-science` 9,960★ MIT (2026-07-12 푸시) — 데이터 과학 프로젝트 구조의 사실상 표준.

**규약만 채택 (스캐폴드 전체 도입은 안 함 — 이미 문서 체계 존재)**:
1. **`data/raw`(불변) → `interim` → `processed` 계층** — "원본은 절대 수정하지 않는다(immutable)" 원칙을 00장 수집 절의 캐시 구조에 적용. 반복 사이클에서 "어느 단계 산출물인지"가 경로로 드러남.
2. **노트북 명명 규약** — `번호-설명` 순번 명명(우리 `00_`~`09_`와 정합)과 "노트북은 탐색·소통용" 역할 규정 — 노트북_서사 §1과 동일 철학의 외부 검증.
3. **분석은 DAG** — "Analysis is a DAG" 원칙: 단계 산출물의 방향성 의존(우리 NB-R5)과 동형.

**스킬화 제안 (사용자 피드백 "MCP나 스킬처럼 쓸 수 있으면")**: 이번에 확정되는 흐름(수집→탐색→전처리→EDA→모델링→해석 + pandera 게이트 + 개정 이력 셀)을 Dacon 허브 `skills/`에 **`eda-workflow` 스킬로 패키징**하면 다음 대회에서 재사용 가능. 구현은 노트북 개정 완료 후(흐름이 실전 검증된 뒤) 별도 작업으로.
