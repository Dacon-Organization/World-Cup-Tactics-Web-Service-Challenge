- P2가 "미확인"으로 남겼던 항목을 **명백한 불일치(사실)**로 확정. CC BY-SA 4.0 표기는 원 라이선스의 오기 또는 무단 재라이선싱.

**② 1차 채택 후보 — `joshfjelstul/world-cup-database`(Kaggle, CC BY-SA 4.0)**
- 원저작자 본인이 GitHub·Kaggle에 동일 선언한 **1차 출처**(재업로드 아님). CC BY-SA 4.0 요건(저작자표시 + 동일조건변경허락) 충족 시 상업적 이용 가능.
- ⚠️ **SA 조항**: 이를 활용한 파생 데이터셋도 CC BY-SA로 재배포 의무 → **코드베이스 라이선스(MIT 등)와 데이터셋 라이선스(CC BY-SA)를 분리 고지** 필요.

**③ 조건부/보수적 취급 — `Mominullptr/fifa-world-cup-2026-dataset`(CC0)**
- 카드 내부는 CC0 자체 선언으로 문구 일치하나, 제작자가 FIFA.com·Sofascore·FBref에서 "소싱"했다고 자체 서술 → **업스트림 재라이선싱 권한 미확인**. CC0 선언 신뢰하되 FIFA 상표·로고·공식 사실 재현 시 별도 검토.

**④ 3개 관할 데이터베이스권 1차 법령 대조 (핵심 설계 원칙)**
- 미국(17 U.S.C. §102(b) + Feist), 한국(저작권법 §93 전문 4개항 + 대법원 2021도1533), EU(Directive 96/9/EC §7)를 각각 1차 원문 확인.
- 공통: **개별 사실 데이터 자체는 3개 관할 모두 미보호**. 차이: 한국·EU는 sui generis 데이터베이스권 인정(미국은 없음).
- **설계 원칙(반영 대상)**: 여러 독립 출처를 대조해 직접 타이핑·재구성하면 위험 낮음. 그러나 **단일 소스에서 반복·체계적으로 추출**하는 순간 한국 §93② 단서·EU §7⑤가 발동해 위험 급상승 → 더미 데이터는 다출처 대조 방식으로 구성.
