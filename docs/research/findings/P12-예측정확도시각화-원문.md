**구현 스케치 (TypeScript, 브라우저 로컬 즉시 실행)**
```ts
// 몬테카를로 진출확률 m/n에 대한 Wilson 95% 신뢰구간
function wilson(m: number, n: number, z = 1.96) {
  const p = m / n;
  const denom = 1 + z*z/n;
  const center = (p + z*z/(2*n)) / denom;
  const half = (z/denom) * Math.sqrt(p*(1-p)/n + z*z/(4*n*n));
  return { p, lower: center - half, upper: center + half };
}
```

### 표 3~4에 대한 서비스 적용 권고 [추정]
- **HOPs는 "여러 시나리오 비교"에 쓸 때 가장 효과적이다.** [추정] Hullman 2015는 단일 변수 고분산에서는 HOPs가 오히려 나쁘고(MAE 9.6 vs 바이올린 4.2), **2·3개 결과 비교(예: 진출 vs 탈락 vs 조1위)에서 압도적으로 우수**(오차 0.11 vs 0.37)함을 보였습니다. 따라서 단일 진출확률 하나에는 아이콘 배열, 여러 시나리오 대조에는 HOPs로 역할을 나누는 것이 근거에 맞습니다. 각 프레임을 몬테카를로 1회 결과로 매핑하고 **400ms/프레임**으로 재생하십시오.

## 이 Space 이전 리서치(P3·P5)와의 정합성 점검
- **명칭 불일치 주의:** RESEARCH-LOG의 P3는 "축구 결과 예측·xG 모델링"으로 등재돼 있으나, 실제 저장된 findings 파일(`P3-모델허브-전이학습-검토.md`)의 내용은 **모델 허브 라이선스·전이학습 검토**로, Dixon-Coles/Elo 정확도 지표는 애초에 다루지 않았습니다. 즉 이번 답변은 P3와 **모순되지 않으며**, P3가 비워둔 예측 정확도 공백을 처음으로 메운 것입니다.
- **P5와의 모순:** 없음. 오히려 P5 섹션 5의 3자 요약(538 NBA "90% 예측→90.48% 실현", HOPs·아이콘 배열)을 **1차 출처로 승격·정정**합니다. 단, P5는 538의 축구 시뮬레이션 반복 횟수를 명시하지 않았는데, 이번에 **월드컵 10,000회(2014)·클럽 20,000회**로 확정했습니다. 또한 P5가 인용한 "90.48%"는 NBA CARM-Elo 사례였고, **축구 쪽 공식 보정 진술은 "32% 예측→약 32% 승리"([538 여자WC](https://fivethirtyeight.com/features/how-our-2019-womens-world-cup-predictions-work/))** 라는 점을 구분해야 합니다.
- **P4(ONNX)와 정합:** Wilson/SE 계산은 순수 JS로 클라이언트에서 즉시 처리되어 온디바이스 제약과 충돌하지 않습니다.
