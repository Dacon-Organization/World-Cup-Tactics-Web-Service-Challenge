## 요약 및 판단

### 채택한 결론
- **D&D UX** (NN/g 1차 출처): grab-handle + hover 커서, 그랩 피드백(고스트/드롭섀도), **~100ms 재배치 애니메이션**, 스냅 마그네티즘(드롭존을 시각 경계보다 넓게), 취소는 mouseup+Esc.
- **접근성** (WCAG 2.2): **SC 2.5.7 Dragging Movements(AA)** — 모든 드래그에 "클릭-클릭" 대안 필수. 키보드 경로(Tab→Space→방향키→Enter→Esc)는 2.5.7과 **독립 요건**. 모바일 손가락 타겟 1cm×1cm, 피드백이 손가락에 안 가리게 오프셋.
- **역할 몰입**: 다이어제틱 UI가 몰입 유의 상승(실험 연구). EA FIFA "목적적 몰입"(기능적/감정적 UI 분리). FM 비평 → **이분법 토글보다 연속 슬라이더가 감독 행위감을 살림**. → 슬라이더 라벨은 "라인 높이/압박 강도" 감독 용어 + "낮음/보통/높음" 질적 라벨 병기.
- **지연 임계값** (Nielsen 1993): **100ms(직접 조작감)/1초(사고 흐름)/10초(주의 유지)**. 슬라이더 시각 프리뷰 100ms 내, 몬테카를로 10초 초과 시 진행률+취소.
- **확률 시각화**: 단일 수치 오독 편향 → 아이콘 배열(100중 62) 병행, HOPs 스타일 애니메이션 옵션, FiveThirtyEight식 시간축 승률 라인차트(리플레이 모드).
- **온보딩**: 점진적 공개(슬라이더 4개+예측 패널만 노출, 세부는 접기) + **훈련용 바퀴(첫 진입 시 강팀 기본값으로 이미 예측치 표시)** → 심사자가 빈 화면 아닌 완성 상태에서 "조작→반응"을 1~2초 내 발견.
