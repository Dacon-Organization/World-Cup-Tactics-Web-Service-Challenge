# 가공명 체계 v1.0 — 생성 규칙·검증 절차·리플레이 연결

> 실격 직결 영역(P7)의 실행 명세입니다. 구성 방식 결정의 배경은 [ADR-005](../../decisions/ADR-005-가공명과_데이터_구성.md) 참조.
> 이 문서로 **미해소 게이트 2건(F01 토큰 표기 · F08 기본 상대)을 해소**합니다.

## 1. 법리 기반 — 왜 "완전 가공명"인가 (P7)

- **서울서부지법 2010카합245** (법제처 원문 로컬 검증 완료, P7): 성명을 **변형해도 특정인이 연상되면** 퍼블리시티권 침해. → PES/FM식 연상형 변형명(Naldorinho류) 관행은 참고 부적합 판정(원장 기록)
- 따라서 3단계 전면 금지: ① 실명 ② 이니셜(H.M. Son류) ③ 연상형 변형(발음·철자 유사)
- 적용 표면: **UI·공유 이미지·OG 카드·클라이언트 JSON·이 문서를 포함한 공개 문서 전부** (P7 — 문서도 노출 표면)
- 실명이 허용되는 유일한 곳: 데이터 생성·학습 파이프라인 내부 (P7·P10 레이어 분리)

## 2. 생성 규칙

1. **형식**: 한국 선수는 한국식 성+이름 2~3음절 완전 창작. 상대 팀(체코·멕시코·남아공) 선수는 해당 언어권 음운 규칙의 창작명 `[설계 결정]`
2. **금지 조합**: 실존 국가대표(최근 2개 대회 소집 명단)·저명 선수와 ① 성+이름 동시 일치 ② 발음 유사(초성 동일+음절수 동일) ③ 등번호·포지션 조합 연상 — 셋 중 하나라도 걸리면 폐기
3. **생성 방식**: 음절 풀에서 무작위 조합 → 검증(§3) → 수동 검수. **결정적(역추적 가능한) 변환 함수 금지** — "실명→가공명" 규칙이 존재하면 그 자체가 연상 경로가 됨 (ADR-005)
4. **문서 단계 표기**: 가공명 확정 전 문서·예시에는 추상 표기(FW-A, MF-B)만 사용 — 이 문서에도 가공명 예시를 싣지 않는다(확정 데이터는 `players.json`이 유일 원본)

## 3. 검증 절차 (빌드 스크립트 — 데이터_설계 §3의 ②)

```
입력: players.json 후보 가공명 목록
대조: 학습 데이터 실명 목록(파이프라인 내부) + 최근 국가대표 소집 명단
검사: ① 완전 일치 0건  ② 초성+음절수 동시 일치 경고 → 수동 재검
      ③ 로마자 표기 유사도(편집거리 ≤2) 경고 → 수동 재검
출력: 통과 목록만 players.json으로 — 경고 잔존 시 빌드 실패
```

수동 검수 기록은 데이터 생성 PR 본문에 남김(심사 대비 감사 추적).

## 4. 리플레이 이벤트 연결 (ADR-005 결정의 실행)

- 실제 이벤트(체코전 67' 득점 등)의 귀속을 위해 **실명↔가공명 1:1 배정표는 데이터 생성 파이프라인 내부에만** 존재. 클라이언트 JSON·화면·공유 카드에는 가공명 ID만 도달
- 배정은 무작위 — 포지션 정합(GK는 GK에)만 유지 `[설계 결정]`
- 이벤트 서술은 사실만: "67' 득점 — {가공명}" (평가 서술 금지, P11·비하 금지)

## 5. 게이트 해소 (기존 [NEEDS CLARIFICATION] 2건 확정)

### 5.1 F01 토큰 표기 → **확정**

- 토큰 = **가공명 성(姓) 1음절 + 포지션 색상 링** (GK/DF/MF/FW 4색)
- 근거: ① 등번호 표기는 연상 리스크(§2-2) ② 성 1음절은 터치 타겟(1cm) 안에서 가독 확보 ③ 색상+텍스트 이중 부호화(색맹 접근성, F12-R3와 동일 원칙)
- 탭/포커스 시 전체 가공명 툴팁 노출
- → F01 문서의 `[NEEDS CLARIFICATION]`은 다음 버전업(v1.1) 때 본 확정으로 치환 (변경노트 누적)

### 5.2 F08·ADR-004 기본 상대 → **확정: 체코 (matchId: cze)**

- 근거: ① 리플레이 1차전 상대 — 첫 진입 전술보드와 리플레이 모드가 같은 상대를 공유해 "공통 퍼즐"(P8) 서사가 이어짐 ② 실제 결과가 2-1 승(P11)이라 기본 화면 확률이 비관적이지 않음(첫인상) ③ 중립 가상 팀 안은 데이터 제작 비용만 늘리고 서사 연결이 없어 기각
- → `defaults.json.opponentMatchId = 'cze'` (데이터_설계 §2.4)
