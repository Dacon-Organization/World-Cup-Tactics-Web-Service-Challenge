# ADR-008. 슬라이더 4종의 모델 반영 — 학습 피처가 아닌 입력 조정 계층

- **Status**: Accepted (2026-07-23) — **ADR-002 Consequences 잔여 과제의 해소**
- **관련**: [피처_정의서_v1_0.md](../ml/version1.0/피처_정의서_v1_0.md) §4~§5 · ADR-002 · F04 · P3

## Context

ADR-002는 슬라이더 4종(라인 높이·압박 강도·공격 폭·템포)을 확정하면서 잔여 과제를 남겼다:
"공격 폭·템포의 모델 기여도는 학습 단계에서 검증 필요 — 기여도가 낮으면 반영 방식을 조정".

이번 설계 착수 전 자체 검증(2026-07-23, gh api로 `jfjelstul/worldcup` 27개 테이블 전수 확인)으로
전제 사실이 확정됐다: **학습 데이터셋에는 전술 성향(포메이션·라인 높이·압박 등) 컬럼이
존재하지 않는다.** 경기·팀·선수·득점·교체·경고 테이블뿐이다. 즉 "슬라이더를 학습 피처로 넣고
기여도를 본다"는 경로는 기여도가 낮은 것이 아니라 **원천적으로 성립하지 않는다**.
학술 쪽도 같다 — 포메이션 상성과 승패를 직접 연결한 재현 가능 연구는 부재가 확인됐다(P3).

남는 선택지는 둘이다: (a) 전술 입력을 규칙 기반으로 모델 입력에 반영하는 조정 계층,
(b) 전술 입력을 예측에서 배제. (b)는 서비스 핵심 경험("전술을 만지면 승부가 바뀐다")과
F04 수용 기준("4종 각각 조작 시 확률이 변한다")을 부정하므로 기각.

## Decision

**슬라이더 4종·포메이션·배치는 학습 피처가 아니라 [입력 조정 계층]으로 반영한다.** `[설계 결정]`

1. **조정 대상**: 결과 확률 모델의 팀 강도 입력(유효 Elo·유효 공/수 지표)과 스코어 모델의
   λ 쌍. 슬라이더는 이 입력들에 유계(bounded) 델타를 가한다.
2. **양날 설계**: 모든 슬라이더는 이득과 리스크를 동시에 가진다(예: 라인 높이↑ = 자기 λ↑ &
   상대 λ↑). 따라서 "전부 최대로 올리면 무조건 유리"가 되는 퇴화 전략이 구조적으로 불가능하다.
   극단값일수록 리스크 항이 비선형으로 커진다. (구체 매핑표: 피처_정의서 §4)
3. **효과 상한 문서화**: 슬라이더 1종의 최대 효과는 사전 정의된 δ 상한을 넘지 않는다.
   상한값은 몬테카를로 민감도 분석으로 캘리브레이션하고 문서에 기록한다.
4. **정직 고지**: 기획서·화면 어디에서도 전술 반응을 "데이터로 학습된 효과"로 서술하지 않는다.
   "역사 데이터로 학습한 팀 강도 모델 + 축구 도메인 지식으로 설계한 전술 반응 규칙"으로
   이원 구조를 명시한다. 포지션 적합도 페널티는 P3 권고대로 [추정] 휴리스틱임을 유지한다.

**기여도 방침(ADR-002 잔여 과제에 대한 답)** — 3단:
① 학습 가능한 전술 프록시(예: 데이터셋에서 유도 가능한 공격성향 지표)가 발견되면 검증 후
피처로 채택 → ② 부재·기여 미미(현재 확인된 상태)면 조정 계층이 확정 경로 → ③ 어느 경우든
효과 상한과 근거 태그([설계 결정]/[추정])를 문서화하고 기획서에 이원 구조를 고지한다.

## Consequences

- (+) ADR-002 잔여 과제가 "검증 대기"에서 "사실 확인에 근거한 설계 확정"으로 종결
- (+) 데이터에 없는 것을 학습했다고 주장하지 않음 — 심사자 예상 질문("전술 효과를 어떻게
  학습했나?")에 대한 답이 선제적으로 문서화됨 (T5 방어)
- (+) 조정 계층은 순수 함수라 단위 테스트 가능 — 슬라이더 조작→확률 변화의 방향·크기를
  결정적으로 검증 (F04 GWT 실측 가능)
- (−) 전술 반응의 크기가 실측 근거 없는 설계값 — δ 상한·민감도 분석으로 완화하되,
  이 한계 자체를 기획서 10절 "모델 한계 정직 고지"에 포함한다
- (−) 조정 계층 로직이 Python(학습 시 검증)과 JS(런타임)에 이중 구현됨 — 동일 케이스
  대조 테스트로 정합 유지 (ML_설계 §5.3 정합 검증에 포함)
