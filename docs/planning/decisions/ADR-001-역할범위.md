# ADR-001. 역할 범위 — 익명 방문자 단일, 관리자 기능 미포함

- **Status**: Accepted (2026-07-22)
- **관련**: 기획서 골격 4절(서비스 개요) 원고에 인용 예정

## Context

선행 팀 프로젝트는 사용자/관리자 기능명세를 분리 운영했고, 이번 프로젝트도 역할 범위를
명시적으로 결정해야 한다는 피드백이 있었다. 대회 조건: "별도 회원가입 또는 유료 결제 없이
심사 가능해야" 하며(대회요강 제출물 세부 조건), 최종 커밋 마감(8/3 10:00) 이후 저장소 수정 불가.

## Decision

**익명 방문자 단일 역할. 회원가입·로그인·관리자 콘솔을 만들지 않는다.**

1. 심사 규칙 "회원가입 없이 심사 가능"과 정합 — 인증이 없으면 인증 실패 계열 버그도 구조적으로 없음
2. 콘텐츠가 정적 JSON(`src/data/`)이므로 운영 수단은 git 커밋 — 별도 관리자 UI가 필요 없고, 저장소 동결(8/3) 이후 콘텐츠 변경이 없다는 대회 조건과도 일치
3. 12일 개발 기간에 관리자 콘솔은 완성도 25점(오류 없이 동작)을 위협하는 범위 확장

사용자 상태(전술 구성)는 서버 저장 없이 URL 인코딩(F13)과 브라우저 로컬로만 유지한다.

## Consequences

- (+) 인증·권한·개인정보 취급 전 영역이 스코프 아웃 → 완성도 집중, 개인정보 리스크 0
- (+) 기획서에 "왜 관리자가 없는가"를 설계 결정으로 선제 답변 가능(심사자 예상 질문 방어)
- (−) 사용자별 전술 저장·히스토리 기능 불가 → URL 공유가 사실상의 저장 수단임을 기획서에 명시
