# ADR-006. 모델 아키텍처 — 2단 구조(결과 확률 + 스코어 생성)와 조건부 2단 샘플링

- **Status**: Accepted (2026-07-23)
- **관련**: [ML_설계_v1_0.md](../ml/version1.0/ML_설계_v1_0.md) §4 · [평가_설계_v1_0.md](../ml/version1.0/평가_설계_v1_0.md) §4 · P3·P12 · F05~F07

## Context

리서치는 모델 후보를 확정했다 — Elo 차이 covariate 로지스틱/더블포아송 baseline + Dixon-Coles τ
저점수 보정 + Karlis-Ntzoufras bivpois 스코어라인 (P12), 그리고 브라우저 제약상 경량
GBDT/로지스틱 우선 (P3·P4). 그러나 **"승/무/패 확률을 내는 분류 모델"과 "스코어라인 분포를 내는
포아송 계열 모델"을 한 서비스 안에서 어떻게 결합하는가**는 리서치가 답하지 않는 설계 재량이다.

제약이 두 방향에서 온다:

1. **F07 수용 기준**: "배열·퍼센트·밴드가 동일 확률을 가리킨다(상호 불일치 0)" — 화면에 표시되는
   단일 추론 확률과 몬테카를로 집계 확률이 어긋나면 수용 기준 위반이다.
2. **기능 요구의 이중성**: F05(승/무/패 3분류 확률)는 분류 모델이면 충분하지만,
   F07 HOPs(프레임=몬테카를로 1회)·F12(시간축 승률)·F11(개입 재추론)은 **스코어라인 수준의
   샘플링**이 있어야 표현이 풍부해진다.

단일 모델로 두 역할을 겸하는 대안(포아송 λ만 학습하고 승/무/패는 λ에서 유도)도 검토했으나,
그 경우 성능 방어선(RPS 0.19~0.22, P12)을 지키는 최적화와 스코어 표현력 최적화가 한 모델에
묶여 서로를 제약한다.

## Decision

**2단 구조를 채택하고, 둘을 조건부 2단 샘플링으로 결합한다.**

1. **결과 확률 모델(주선)** — 승/무/패 3-class 분류.
   LightGBM·XGBoost·CatBoost + 로지스틱 기준선(Elo 차이 covariate, P12) → Optuna(목적함수 RPS)
   → 검증 RPS 기반 가중 소프트 보팅 앙상블 → onnxmltools 변환 → ORT-Web wasm 단일 추론.
   **성능 방어선(RPS 0.19~0.22 / 정확도 0.50~0.55, P12)의 평가 대상은 이 모델이다.**
2. **스코어 생성 모델(보조)** — bivpois 골격의 기대득점(λ 쌍) 회귀 + Dixon-Coles τ 저점수 보정.
   계수를 경량 JSON으로 내보내 JS에서 포아송 샘플링한다(ONNX 불필요 — 닫힌 수식).
3. **결합 = 조건부 2단 샘플링** — 몬테카를로 1회는
   ① 결과 범주 ~ Categorical(p) (p = ①의 ONNX 추론값)
   ② 스코어라인 ~ P(score | 결과 범주, λ) (결과 범주와 일치하는 스코어만 남긴 절단 분포)
   의 2단으로 뽑는다. 집계 확률 p̂은 **구성상 p로 수렴**하므로 F07 "상호 불일치 0"이
   수학적으로 보장된다. (수렴 논증: [평가_설계_v1_0.md](../ml/version1.0/평가_설계_v1_0.md) §4)

## Consequences

- (+) 표시 확률(단일 추론)과 몬테카를로 집계가 원리적으로 일치 — F07 수용 기준을 구현 품질이
  아니라 **구조로** 충족. 밴드는 순수하게 시뮬레이션 표본 오차만 반영한다
- (+) 방어선 평가(3-class RPS)와 스코어 표현력(HOPs·타임라인)을 각자 최적화 — 한쪽의 타협이
  다른 쪽을 오염시키지 않음
- (+) 스코어 모델은 닫힌 수식이라 ONNX 변환 대상이 아님 — 변환 실패 리스크가 주선 1개로 축소
- (−) 조건부 절단 분포의 정규화(범주별 스코어 확률 재분배)를 JS로 구현해야 함 — 구현 복잡도
  증가. 완화: 스코어 상한(예: 팀당 0~6골)을 두면 유한 격자 계산으로 단순화 `[설계 결정]`
- (−) 두 모델이 크게 어긋나면(예: 분류는 승 60%인데 λ는 열세) 조건부 스코어가 부자연스러워질
  수 있음 — 학습 단계에서 두 모델의 유도 확률 괴리를 정합 지표로 관리 (평가_설계 §2.4)
