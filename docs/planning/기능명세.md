> 절 구성은 [기획서-골격.md](기획서-골격.md), 화면·흐름은 [화면흐름.md](화면흐름.md)를 참조합니다.

> - 수치·주장 뒤 `(Pn)` = [리서치 원장](../research/RESEARCH-LOG.md) `회수됨` 항목으로 역추적

## P0 — 전술보드 + 온디바이스 예측

### F1. 전술보드 피치 + 포메이션 프리셋

- **설명**: 축구 피치 위에 11명(GK 1 + 필드 10)을 배치. 포메이션 프리셋 선택 시 표준 위치로 자동 배치.
- **프리셋**: 4-3-3 / 4-4-2 / 3-4-3 / 3-5-2 `[설계 결정]` — 대회요강 원문 인용 가능("4-3-3이 아니라 3-4-3으로 가야지")
- **스펙**:
  - 프리셋 전환 시 선수들이 새 위치로 ~100ms 애니메이션 이동 (P5 재배치 애니메이션 준용)
  - 피치는 자체 SVG — FIFA IP 요소 없음 (P7)
- **수용 기준**:
  - [ ] 프리셋 4종 각각 선택 시 11명이 표준 위치로 이동한다
  - [ ] 프리셋 전환 후에도 개별 배치 수정(F2)이 가능하다
  - [ ] 모바일 세로 화면에서 피치 전체가 잘리지 않고 표시된다

### F2. 선수 드래그앤드롭 (NN/g 패턴, P5)

- **설명**: 선수 토큰을 드래그해 피치 위 임의 위치로 이동.
- **스펙** (전부 P5·NN/g 1차 출처):
  - 아이들: grab-handle 어포던스 + hover 시 `grab` 커서
  - 드래그 중: 고스트(반투명 원본 잔상) + 드롭섀도로 "들려 있음" 표현
  - 드롭: ~100ms 재배치 애니메이션
  - 스냅 마그네티즘: 드롭존(포지션 슬롯)을 시각 경계보다 넓게 잡아 근접 드롭 허용
  - 취소: 피치 밖 mouseup = 원위치 복귀, 드래그 중 **Esc = 취소**
  - 모바일: 터치 타겟 1cm×1cm 이상, 드래그 피드백은 손가락에 가리지 않게 오프셋
- **수용 기준**:
  - [ ] 데스크톱 마우스·모바일 터치 모두에서 드래그로 선수를 이동할 수 있다
  - [ ] 드래그 중 Esc를 누르면 선수가 원위치로 복귀한다
  - [ ] 드롭존 근처에 놓으면 슬롯에 스냅된다
  - [ ] 이동 완료 100ms 내에 시각 피드백이 끝난다

### F3. 클릭-클릭 대안 + 키보드 경로 (WCAG 2.5.7 AA, P5)

- **설명**: 드래그가 어려운 사용자를 위한 완전 대체 조작 경로. **모든 드래그 기능에 필수**.
- **스펙**:
  - 클릭-클릭: 선수 탭(선택 하이라이트) → 목적지 슬롯 탭(이동). 다른 선수 탭 시 선택 전환
  - 키보드: Tab(포커스 이동) → Space(선택) → 방향키(위치 이동) → Enter(확정) → Esc(취소) — 2.5.7과 **독립 요건** (P5)
- **수용 기준**:
  - [ ] 마우스 드래그 없이 클릭(탭)만으로 선수 이동이 완료된다
  - [ ] 키보드만으로 선수 이동이 완료된다
  - [ ] 선택 상태가 시각적으로 명확히 구분된다

### F4. 전술 슬라이더 4개 (P5)

- **설명**: 팀 전술 성향을 연속값으로 조정. **이분법 토글이 아닌 연속 슬라이더** — 감독 행위감의 근거 (P5, FM 비평).
- **슬라이더 구성**: 라인 높이 · 압박 강도 (P5 라벨 예시 채택) + 공격 폭 · 템포 `[설계 결정]`
- **스펙**:
  - 라벨: 감독 용어("라인 높이") + 질적 라벨("낮음/보통/높음") 병기 (P5)
  - 조작 즉시(≤100ms) 피치 위 시각 프리뷰(예: 라인 높이 → 수비 라인 위치 표시) (P5)
  - 슬라이더 값은 모델 입력 피처로 연결 (ML 파이프라인 — 기획서 10절)
- **수용 기준**:
  - [ ] 슬라이더 조작 100ms 내 피치에 시각 변화가 나타난다
  - [ ] 4개 슬라이더 모두 예측 결과(F5)에 영향을 준다
  - [ ] 터치로 조작 가능한 크기(1cm 이상)다

### F5. 온디바이스 예측 패널 (P4)

- **설명**: 현재 전술(배치+슬라이더)에 대한 승/무/패 확률을 **심사자의 브라우저 안에서** 추론. 서버 API·키 0개.
- **스펙** (전부 P4):
  - 런타임: ONNX Runtime Web **wasm 전용 서브셋**(`ort.wasm.min.js`, raw-min 약 47KB)
  - 모델: 직접 학습 경량 GBDT/로지스틱 → onnxmltools 변환(TreeEnsemble). 차선책: m2cgen 순수 JS 생성
  - 추론은 **Web Worker**에서 실행 — 메인 스레드(D&D·슬라이더) 비블로킹
  - 모델·wasm은 Next.js `public/` + Vercel 직접 서빙(CORS 무문제) + Service Worker 캐싱
- **수용 기준**:
  - [ ] 네트워크 차단 상태(재방문)에서도 추론이 동작한다
  - [ ] 추론 중에도 D&D·슬라이더 조작이 끊기지 않는다
  - [ ] 외부 API 호출이 0건이다 (DevTools Network로 검증)

### F6. 몬테카를로 시뮬레이션 (P12)

- **설명**: 단일 추론 확률을 기반으로 경기 시뮬레이션 반복 → 확률 분포·신뢰구간 산출.
- **스펙**:
  - **빠른 모드 5,000회(기본) / 정밀 모드 25,000회(옵션)** (P12 — 538 클럽 20,000·Opta 25,000 정합)
  - 정밀 모드가 10초 초과 가능 → **진행률 표시 + 취소 버튼 필수** (P5 10초 임계)
  - Web Worker에서 실행, 취소 시 즉시 중단
- **수용 기준**:
  - [ ] 빠른 모드가 기본이며 조작 후 자동 실행된다
  - [ ] 정밀 모드 실행 중 진행률이 표시되고 취소가 즉시 반영된다
  - [ ] 두 모드의 결과에 반복 횟수가 명시된다

### F7. 확률 시각화 (P12)

- **설명**: 단일 수치 오독 편향을 막는 다층 확률 표현.
- **스펙** (전부 P12):
  - **10×10 아이콘 배열**: top/row 채움 방식, central/random 편향 배치 회피, 퍼센트 숫자 병기, 스크린리더 텍스트 라벨
  - **Wilson score 신뢰구간**: 0%/100% 근접에서도 [0,1] 내부 보장, 순수 JS 즉시 계산. SE = √(p̂(1−p̂)/N)
  - **HOPs 옵션**: 400ms/프레임(사전 생성 프레임 재생, <100ms 금지). 각 프레임 = 몬테카를로 1회. 추론과 분리(사전 렌더)
- **수용 기준**:
  - [ ] 승/무/패가 아이콘 배열 + 퍼센트 + 밴드로 동시에 표시된다
  - [ ] 스크린리더가 확률을 텍스트로 읽을 수 있다
  - [ ] HOPs 재생/정지 토글이 동작한다

### F8. 온보딩 기본 상태 (P5)

- **설명**: **빈 화면 금지.** 첫 진입 시 기본 전술과 예측치가 이미 표시된 완성 상태로 시작.
- **스펙**:
  - 기본 전술: 4-3-3 + 슬라이더 중앙값 `[설계 결정]` → 진입 즉시 예측 패널에 확률 표시
  - 점진적 공개: 1차 노출은 슬라이더 4개 + 예측 패널만, 세부 설정은 접기 (P5)
  - 심사자가 1~2초 내 "조작→반응" 구조를 발견하는 것이 목표 (P5)
- **수용 기준**:
  - [ ] 첫 진입 화면에 이미 배치·예측치가 표시되어 있다 (빈 상태 화면 부재)
  - [ ] 모델 로딩 완료 전에는 스켈레톤+기본 통계가 먼저 보인다

### F10. 이벤트 타임라인 (P11)

- **설명**: 경기 진행을 분 단위 이벤트(득점·교체·경고)로 재생.
- **스펙**:
  - 이벤트 분(分)은 **P11 확정 이벤트 목록만** 사용 (체코전 67'·80' 득점, 남아공전 63' 실점, 46' 교체 등)
  - xG는 **"Opta xG" 출처 라벨 병기** — 남아공전 1.16/0.90 (정밀값, P11)
  - **남아공전 경고 이벤트는 P13(FIFA 집계 1 vs 위키 개별 2 상충) 확정 전 미표시** (P11)

  - [ ] 미확정(P13 대기) 이벤트가 화면에 없다

### F11. "내가 감독이라면" 개입

- **설명**: 타임라인의 특정 시점을 선택해 전술을 바꾸고, 재예측 결과를 실제 결과와 나란히 비교.
- **스펙**:
  - 개입 시점 선택 → 전술보드(F2~F4 재사용)로 변경 → 재추론(F5·F6) → 실제 결과와 비교 패널
  - 비교 서술은 **"다른 선택지가 있었다" 관점만** — 실제 선택을 "잘못"으로 규정하는 표현 금지 (비하 금지, 카피 단위 관리)
  - 예측은 확률로만 제시 — "이겼을 것" 단정 금지 (골격 10절 정직 고지 원칙)
- **수용 기준**:
  - [ ] 시점 선택 → 전술 변경 → 재예측 → 비교가 한 흐름으로 이어진다
  - [ ] 비교 화면의 모든 카피가 비하 금지 검수를 통과한다
  - [ ] 결과가 단정이 아닌 확률(밴드 포함)로 표시된다

### F12. 시간축 승률 라인차트 (P5)

- **스펙**: 이벤트(F10) 발생 지점에 마커, 개입(F11) 시 가상 라인을 실제 라인과 병렬 표시
- **수용 기준**:
  - [ ] 실제 경기 라인과 개입 시나리오 라인이 시각적으로 구분된다
  - [ ] 이벤트 마커 탭 시 해당 이벤트 정보가 표시된다

## P2 — 공유

### F13. 전술 상태 URL 인코딩 (P8)

- **설명**: 포메이션+배치+슬라이더 상태를 URL로 인코딩. **수신자 브라우저에서 재추론** — 공유에 서버 불필요.
- **스펙**:
  - 인코딩 대상: 포메이션 ID + 11명 좌표(양자화) + 슬라이더 4값 + (리플레이면) 경기 ID·개입 시점
  - URL 길이가 카톡·SNS에서 잘리지 않는 한계선은 **P17 실측 대기** — 구현 시 확인
- **수용 기준**:
  - [ ] 공유 URL을 새 브라우저에서 열면 동일한 전술이 복원된다
  - [ ] 복원된 화면에서 재추론이 자동 실행된다

### F14. 동적 OG 이미지 (P8)

- **설명**: 공유 URL마다 전술 카드를 OG 이미지(1200×630)로 서버 생성.
- **스펙**:
  - Next.js `opengraph-image.tsx` + `ImageResponse` (satori 내장 — 직접 의존 불필요) (P8)
  - **카카오톡은 OG 스크래핑만으로 미리보기 성립 — 앱 키 불필요** ("심사자 별도 키 없이" 규칙 정합) (P8)
  - 카톡 스크랩 캐시 존재 → 시연 전 카카오 공유 디버거로 캐시 갱신 (체크리스트 이관, P8)
  - 이미지 내 표기: 가공명·자체 SVG·flag-icons만 (P7 전면 적용)
- **수용 기준**:
  - [ ] 공유 URL을 카톡에 붙여넣으면 전술 카드 미리보기가 뜬다 (앱 키 없이)
  - [ ] OG 이미지에 실명·FIFA IP 요소가 0건이다

### F15. 공유 실행 3계층 (P8)

- **설명**: 환경별 최적 공유 수단으로 자동 폴백.
- **스펙**:
  - 1계층: URL 복사 (기본, 전 환경)
  - 2계층: **Web Share API** — `navigator.share` **feature detection 필수** (Firefox 데스크톱 미지원). HTTPS + 사용자 제스처 요건 (P8)
  - 3계층: **클립보드 이모지 그리드** — 승률을 이모지 행렬로 축약(Wordle 방식), `navigator.clipboard.writeText` (P8)
  - 이모지 그리드는 F7의 10×10 배열을 텍스트로 번역 — 동일 포맷 (P8·P12 정합)
- **수용 기준**:
  - [ ] 모바일에서 OS 공유 시트가 열린다
  - [ ] Web Share 미지원 브라우저에서 클립보드 폴백이 동작한다
  - [ ] 이모지 그리드가 스코어 스포일러 없이 확률 구조만 노출한다

### F16. 내 전술 카드 (P8)

- **설명**: 공유되는 시각 결과물. Wrapped 원칙 — **데이터 스토리 1개당 카드 1개**.
- **스펙**:
  - 구성: 포메이션 미니맵 + 슬라이더 설정 + 승/무/패 확률 밴드 + **경기 식별자(상대 국기+라운드) 필수** — 공통 퍼즐 비교 가능성의 핵심 (P8)
  - 카드 카피도 비하 금지 검수 대상 — "감독님의 선택은 달랐다" O / 특정 인물 "잘못" 규정 X (P8)
- **수용 기준**:
  - [ ] 카드만 보고 "어느 경기, 어떤 전술, 어떤 확률"이 읽힌다
  - [ ] 같은 경기의 다른 전술 카드와 비교 가능한 동일 레이아웃이다
