# F08. 온보딩 기본 상태 (빈 화면 금지)

> **우선순위**: P0 · **담당 화면**: S1 · **근거**: P5·ADR-004 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "처음 온 사용자는 빈 보드가 아니라, 이미 짜인 전술과 예측을 보며 '만지면 바뀌겠구나'를 1~2초 만에 안다."
- **Why this priority**: 심사자의 첫 10초를 결정 — 튜토리얼 없이 조작→반응 구조를 발견하게 하는 P5 확정 전략.
- **Independent Test**: 시크릿 창 첫 진입만으로 단독 시연 가능(배치+예측치가 이미 표시되어 있는가).

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: 없음(첫 진입 자체가 대상)
- **Post-condition**: 기본 전술(ADR-004: 4-3-3+중앙값) + 예측치 표시 상태. 모든 조작 가능.

## 3. UI/UX·네비게이션

- 점진적 공개: 1차 노출 = 피치 + 슬라이더 4개 + 예측 패널만. 세부(프리셋 변경·정밀 모드·HOPs)는 접힘 (P5)
- 별도 튜토리얼·코치마크 없음 `[설계 결정]` — "완성 상태에서 발견"이 P5 채택 전략이므로 오버레이가 오히려 방해
- 로딩 순서: 스켈레톤 → 기본 배치·기본 통계 즉시 → 모델 준비 완료 시 추론값으로 자연 갱신

## 4. 상세 로직·설계 근거

- 기본 통계: 모델 로딩 전에도 표시할 사전 계산 확률(빌드 타임 산출) `[설계 결정]` — F05-R5와 연동
- 기본 상대: `[NEEDS CLARIFICATION: 체코 vs 중립 가상 팀 — 데이터 설계의 가공명 체계와 함께 확정 (ADR-004)]`
- **설계 근거**: "훈련용 바퀴(기본값으로 이미 예측치 표시)" P5 확정

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F08-R1 | WHEN | **WHEN** 첫 진입하면, 시스템은 기본 전술 배치와 확률(사전 계산 또는 추론)을 즉시 표시한다 |
| F08-R2 | Ubiquitous | 시스템은 어떤 시점에도 "빈 보드" 상태를 표시하지 않는다 |
| F08-R3 | WHEN | **WHEN** 모델 로드가 완료되면, 시스템은 사전 계산 확률을 실제 추론값으로 무단절 교체한다 |
| F08-R4 | IF-THEN | **IF** 초기 로딩이 3초를 초과하면, **THEN** 시스템은 스켈레톤 대신 기본 배치+사전 계산 통계를 우선 표시한다(체감 대기 제거) |
| F08-R5 | IF-THEN | **IF** JS 실행이 실패하면(치명 오류), **THEN** 시스템은 서버 렌더된 기본 배치 정적 화면과 안내 문구를 표시한다(백지 화면 금지) |
| F08-R6 | IF-THEN | **IF** 공유 URL(F13)로 진입하면, **THEN** 시스템은 기본 상태가 아닌 URL 복원 상태를 우선한다(F08은 무파라미터 진입에만 적용) |

## 6. 시나리오·예외

- **Happy Path**: 첫 진입 → 배치+확률 즉시 표시 → 1~2초 내 슬라이더 발견 → 조작 → 반응
- **Edge Cases** (물음표 → 해소 R):
  - 느린 회선에서 모델이 늦으면? → 사전 계산값 우선, 완료 시 교체 (F08-R4·R3)
  - JS가 죽으면 백지가 뜨나? → SSR 정적 폴백 (F08-R5)
  - 공유 링크로 들어왔는데 기본값이 덮어쓰나? → URL 상태 우선 (F08-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 시크릿 창 **When** 첫 진입 **Then** 3초 내 배치와 확률이 보인다(빈 보드 구간 없음)
- **Given** 네트워크 스로틀(3G) **When** 진입 **Then** 스켈레톤→기본값→추론값 순서로 표시되며 백지 구간이 없다
- **Given** 첫 화면 **Then** 노출 요소가 피치·슬라이더 4개·예측 패널로 제한된다(세부는 접힘)
