# F16. 내 전술 카드

> **우선순위**: P2 · **담당 화면**: S1·S2b·S3(+F14 재현) · **근거**: P8·P7 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 자기 전술과 그 확률이 담긴 카드 한 장을 받는다 — 남과 비교하고 싶어지는."
- **Why this priority**: Wrapped 원칙 "데이터 스토리 1개당 공유 카드 1개"(P8) — 공유 동기의 시각적 실체. 카드 없이는 3계층(F15)이 빈손이다.
- **Independent Test**: 카드만 보고 "어느 경기·어떤 전술·어떤 확률"이 읽히는지 제3자 확인만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: 전술 상태 + 시뮬레이션 결과(F06) 존재
- **Post-condition**: 카드 렌더 완료 — 화면 표시용(공유 전 미리보기)과 OG 재현(F14)의 공통 사양

## 3. UI/UX·네비게이션

카드 구성(P8 확정):
1. 포메이션 미니맵(피치+토큰 축약)
2. 슬라이더 설정 4종(아이콘+질적 라벨)
3. 승/무/패 확률 밴드(F07 축약형)
4. **경기 식별자(상대 국기+라운드) — 필수** ("같은 멕시코전, 다른 전술" 비교 가능성의 핵심, P8 공통 퍼즐)
5. 서비스명 워터마크(자체 브랜드만)

## 4. 상세 로직·설계 근거

- 화면 카드와 OG 카드(F14)는 동일 레이아웃 사양 공유 — 이중 구현이되 단일 디자인 명세 `[설계 결정]`
- 같은 경기 카드끼리 비교 가능하도록 레이아웃 고정(변형 금지)
- 카드 카피 규칙: "감독님의 선택은 달랐다" 방향만 — 특정 인물 판단을 "잘못"으로 규정 금지 (P8·비하 금지)
- **설계 근거**: 공유 5요소 중 ②드라마 압축·③공통 퍼즐의 시각 구현 (P8)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F16-R1 | WHEN | **WHEN** 공유 흐름이 시작되면, 시스템은 미니맵·슬라이더·확률 밴드·경기 식별자가 포함된 카드를 미리보기로 표시한다 |
| F16-R2 | Ubiquitous | 시스템은 카드에 경기 식별자(상대 국기+라운드)를 항상 포함한다 |
| F16-R3 | Ubiquitous | 시스템은 카드에 실명·이니셜·연상 표기·FIFA IP를 절대 포함하지 않는다 (P7 전면 적용) |
| F16-R4 | IF-THEN | **IF** 리플레이 컨텍스트가 없는 자유 전술이면, **THEN** 시스템은 경기 식별자 자리에 "자유 전술" 배지를 표시한다(빈 영역 금지) |
| F16-R5 | IF-THEN | **IF** 시뮬레이션 결과가 아직 없으면(조작 직후), **THEN** 시스템은 공유 실행을 결과 도착까지 지연하고 로딩 상태를 카드에 표시한다(미완성 카드 공유 금지) |
| F16-R6 | IF-THEN | **IF** 카드 미리보기 렌더가 실패하면, **THEN** 시스템은 카드 없이 URL 공유(F15 계층 1)로 강등하고 안내한다 |

## 6. 시나리오·예외

- **Happy Path**: 멕시코전 개입 후 공유 → 카드 미리보기(국기+라운드 포함) → 시트 공유
- **Edge Cases** (물음표 → 해소 R):
  - 리플레이가 아닌 자유 전술 공유면 식별자는? → "자유 전술" 배지 (F16-R4)
  - 확률 계산 전에 공유를 누르면? → 결과 대기 후 완성 카드만 (F16-R5)
  - 카드 렌더가 깨지면 공유가 막히나? → URL 공유로 강등 (F16-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 완성 카드 **When** 제3자에게 카드만 제시 **Then** 어느 경기·어떤 전술 성향·어떤 확률인지 설명 없이 읽힌다
- **Given** 같은 경기의 두 카드 **Then** 레이아웃이 동일해 나란히 비교 가능하다
- **Given** 생성된 모든 카드 **Then** 실명·FIFA IP 0건, 카피가 비하 금지 검수를 통과한다

## 8. 데이터·상태·비기능

- **연계**: 시각 요소는 F01(미니맵)·F04(슬라이더)·F07(밴드) 축약 재사용 — 신규 시각 문법 없음
- **비기능**: 화면/OG 동일 사양 유지(디자인 명세 단일 출처) · 카드 전체가 P7 라이선스 경계 내 에셋만 사용
