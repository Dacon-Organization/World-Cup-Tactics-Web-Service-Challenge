# F01. 전술보드 피치 + 포메이션 프리셋

> **우선순위**: P0 · **담당 화면**: S1 · **근거**: P5·P7·ADR-003 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 축구 피치 위에서 포메이션을 선택하고, 그 위에 11명의 선수가 배치된 것을 본다."
- **Why this priority**: 서비스의 물리적 무대 — 모든 P0 기능(D&D·슬라이더·예측)이 이 위에서 일어난다.
- **Independent Test**: 프리셋 4종을 각각 선택해 11명이 표준 위치로 이동하는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: S1 진입 완료. 기본 상태(ADR-004: 4-3-3 + 슬라이더 중앙값)로 초기화됨.
- **Post-condition**: 선택한 프리셋의 표준 좌표로 11명 배치. 이후 개별 수정(F02) 가능 상태.

## 3. UI/UX·네비게이션

- 피치: **자체 제작 SVG** (세로 방향, 모바일 기준) — FIFA IP 요소 없음 (P7)
- 선수 토큰: 포지션 약어 + 등번호 없는 추상 토큰 `[NEEDS CLARIFICATION: 토큰 표기(가공명/포지션 약어)는 데이터 설계의 가공명 체계와 함께 확정]`
- 프리셋 선택 UI: 세그먼트 컨트롤 4종(4-3-3 / 4-4-2 / 3-4-3 / 3-5-2, ADR-003) — 접힌 세부 설정 영역에 배치 (P5 점진적 공개)
- 라우팅: `/` (S1 내 구성 요소, 페이지 전환 없음)

## 4. 상세 로직·설계 근거

- 프리셋 = 포지션 슬롯 11개의 정규화 좌표 세트(피치 비율 기준 0~1) — 화면 크기 무관 동일 배치
- 프리셋 전환 시 기존 개별 수정 위치는 폐기하고 표준 좌표로 재배치 `[설계 결정]` — 전환 의도는 "다시 시작"이 자연스러움
- 전환 애니메이션 ~100ms (P5 재배치 애니메이션 준용)
- **설계 근거**: 4종 선정 사유는 ADR-003 (대회요강 원문 예시 2종 포함 + QA 비용)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F01-R1 | WHEN | **WHEN** 사용자가 프리셋을 선택하면, 시스템은 11명 토큰을 해당 표준 좌표로 ~100ms 애니메이션으로 이동시킨다 |
| F01-R2 | WHILE | **WHILE** 프리셋 전환 애니메이션 진행 중, 시스템은 토큰 드래그 시작을 무시한다(진행 중 상태 충돌 방지) |
| F01-R3 | Ubiquitous | 시스템은 항상 피치와 11명 토큰 전체를 뷰포트 내에 렌더링한다(잘림 없음, 모바일 세로 기준) |
| F01-R4 | IF-THEN | **IF** 뷰포트 폭이 320px 미만이면, **THEN** 시스템은 피치를 최소 지원 폭으로 고정하고 가로 스크롤 없이 축소 렌더링한다 |
| F01-R5 | IF-THEN | **IF** SVG 렌더링이 실패하면(비정상 브라우저), **THEN** 시스템은 단순 그리드 폴백 레이아웃으로 토큰 위치를 표시한다 |
| F01-R6 | IF-THEN | **IF** 화면 회전(orientation change)이 발생하면, **THEN** 시스템은 정규화 좌표를 유지한 채 새 뷰포트에 재렌더링한다(배치 정보 손실 없음) |

## 6. 시나리오·예외

- **Happy Path**: 진입(4-3-3 기본) → 프리셋 "3-4-3" 선택 → 11명이 100ms 내 재배치 → 개별 수정(F02) 진행
- **Edge Cases** (물음표 → 해소 R):
  - 프리셋 전환 중에 또 다른 프리셋을 연타하면? → 마지막 선택만 적용, 진행 중 드래그는 무시 (F01-R2)
  - 아주 좁은 화면(폴더블 커버 디스플레이)에서 피치가 잘리면? → 최소 폭 고정 축소 (F01-R4)
  - 가로 모드로 돌리면 배치가 깨지나? → 정규화 좌표 재렌더 (F01-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 기본 상태(4-3-3) **When** 3-4-3 프리셋 선택 **Then** 11명 전원이 3-4-3 표준 좌표에 있고 애니메이션이 완료돼 있다
- **Given** 임의 프리셋 **When** 개별 토큰을 수정(F02) 후 다른 프리셋 선택 **Then** 수정이 초기화되고 새 표준 좌표로 배치된다
- **Given** 모바일 세로 뷰포트(375px) **When** S1 렌더 **Then** 피치·토큰·프리셋 UI가 가로 스크롤 없이 표시된다

## 8. 데이터·상태·비기능

- **데이터**: 프리셋 좌표 세트 4종 — `src/data/formations.json` (정적, 정규화 좌표)
- **상태(Zustand)**: `formation`(프리셋 ID), `positions[11]`(정규화 좌표) — F13 URL 인코딩 대상
- **비기능**: 전환 애니메이션 ~100ms (P5) · 피치 SVG는 자체 제작(P7 라이선스 경계) · 프리셋 컨트롤 터치 타겟 ≥1cm (P5)
