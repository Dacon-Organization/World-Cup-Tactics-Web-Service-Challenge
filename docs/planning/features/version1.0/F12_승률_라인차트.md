# F12. 시간축 승률 라인차트

> **우선순위**: P1 · **담당 화면**: S2b · **근거**: P5·P12 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 경기 시간에 따라 승률이 어떻게 출렁였는지, 그리고 자신의 개입이 그 곡선을 어디서 갈라놓는지 본다."
- **Why this priority**: FiveThirtyEight식 시간축 승률(P5)은 리플레이 모드의 서사 장치 — 개입(F11)의 효과를 한눈에 보이게 한다.
- **Independent Test**: 실제 라인 1개 + 개입 시나리오 라인 1개가 시각적으로 구분 표시되는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: F10 이벤트 데이터 로드 완료. (시나리오 라인은 F11 개입 확정 시)
- **Post-condition**: 실제 라인 렌더, 개입 시 가상 라인 병렬 렌더, 이벤트 마커 활성

## 3. UI/UX·네비게이션

- X축: 경기 분(0~90+) / Y축: 승률(%) — 모바일 폭 기준 축약 눈금
- 실제 라인(실선)과 시나리오 라인(점선+다른 색) — 색만으로 구분하지 않음(패턴 병행, 접근성)
- 이벤트 마커(F10 노드와 동일 지점): 탭 시 해당 이벤트 정보
- 개입 분기점: 시나리오 라인이 갈라지는 지점에 마커

## 4. 상세 로직·설계 근거

- 실제 라인: 이벤트 시점별 사전 계산 승률(빌드 타임 산출, `matches.json`에 포함) `[설계 결정]` — 런타임 계산 아님(완성도 우선)
- 시나리오 라인: F11 재예측 결과에서 생성 — 개입 분 이후 구간만
- 라인 데이터는 확률이므로 밴드(영역 음영) 표시 옵션 (P12 Wilson과 동일 원칙)
- **설계 근거**: FiveThirtyEight식 시간축 승률 라인차트 (P5)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F12-R1 | WHEN | **WHEN** S2b가 로드되면, 시스템은 실제 경기 승률 라인과 이벤트 마커를 렌더한다 |
| F12-R2 | WHEN | **WHEN** 개입(F11)이 확정되면, 시스템은 개입 분부터 시나리오 라인을 실제 라인과 병렬 표시한다 |
| F12-R3 | Ubiquitous | 시스템은 두 라인을 색+선 패턴(실선/점선)으로 이중 구분한다(색맹 접근성) |
| F12-R4 | IF-THEN | **IF** 마커를 탭하면, **THEN** 시스템은 해당 이벤트 정보를 표시하고 다른 열린 정보는 닫는다(F10-R6 동일 원칙) |
| F12-R5 | IF-THEN | **IF** 개입이 취소·변경되면, **THEN** 시스템은 기존 시나리오 라인을 제거(또는 교체)하고 잔상을 남기지 않는다 |
| F12-R6 | IF-THEN | **IF** 차트 폭이 모바일 최소 폭이면, **THEN** 시스템은 눈금을 축약(15분 단위)하되 이벤트 마커는 전부 유지한다 |

## 6. 시나리오·예외

- **Happy Path**: S2b 진입 → 실제 라인 확인 → 63' 실점 마커 탭 → 60' 개입 → 점선 분기 확인
- **Edge Cases** (물음표 → 해소 R):
  - 개입을 여러 번 바꾸면 라인이 쌓이나? → 항상 교체, 잔상 금지 (F12-R5)
  - 좁은 화면에서 마커가 겹치면? → 눈금 축약+마커 유지 (F12-R6)
  - 색맹 사용자가 두 라인을 구분 못 하면? → 실선/점선 이중 구분 (F12-R3)

## 7. 수용 기준 (Given-When-Then)

- **Given** S2b **Then** 실제 라인·마커가 이벤트 데이터와 일치 지점에 렌더된다
- **Given** 개입 확정 **When** 차트 확인 **Then** 분기점 이후 두 라인이 병렬이며 흑백 인쇄로도 구분된다
- **Given** 개입 재변경 **Then** 시나리오 라인이 1개만 존재한다

## 8. 데이터·상태·비기능

- **데이터**: `matches.json.winProbTimeline[]`(사전 계산) + F11 재예측 산출(런타임)
- **비기능**: 순수 SVG(F07과 동일 무의존 원칙) `[설계 결정]` · 마커 터치 타겟 ≥1cm · 실선/점선 이중 구분(접근성)
