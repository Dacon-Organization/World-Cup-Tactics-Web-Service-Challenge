# F05. 온디바이스 예측 패널

> **우선순위**: P0 · **담당 화면**: S1(S2b·S3 재사용) · **근거**: P4 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자가 전술을 바꾸면, 승/무/패 확률이 심사자의 브라우저 안에서 즉시 다시 계산된다."
- **Why this priority**: 서비스의 참신성 핵심(P1 시장 공백: 배치와 결합된 예측) + "심사자 별도 키 없이" 규칙을 구조로 충족하는 장치.
- **Independent Test**: 네트워크 탭을 열어 외부 API 호출 0건인 채로 확률이 갱신되는 것을 보이는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: 모델·wasm 자산 로드 완료(또는 로딩 중 폴백 상태). 전술 상태(F01~F04) 존재.
- **Post-condition**: 승/무/패 확률이 패널에 표시되고 시각화(F07)에 전달됨.

## 3. UI/UX·네비게이션

- 위치: 모바일 하단 고정 패널 / 데스크톱 우측 하단 (화면흐름 S1)
- 표시: 승/무/패 3분류 확률 + 갱신 인디케이터(계산 중에도 직전 값 유지 — 화면 비우지 않음, P5)
- 폴백 상태 표시: 통계 폴백 작동 시 "간이 추정" 배지 `[설계 결정]` — 정직 고지 원칙

## 4. 상세 로직·설계 근거

전부 P4 확정:
- 런타임: ONNX Runtime Web **wasm 전용 서브셋**(`ort.wasm.min.js`, raw-min 약 47KB), TreeEnsemble 연산자
- 모델: 직접 학습 경량 GBDT/로지스틱 → onnxmltools 변환. 차선책 m2cgen 순수 JS 생성
- 실행: **Web Worker** — 메인 스레드(D&D·슬라이더) 비블로킹
- 자산: Next.js `public/` + Vercel 직접 서빙, Service Worker Cache API 캐싱
- **설계 근거**: 파인튜닝 노선 기각("라이선스는 파인튜닝으로 세탁되지 않는다", P9)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F05-R1 | WHEN | **WHEN** 전술 상태가 커밋되면(F02·F03·F04), 시스템은 Worker에서 추론을 실행하고 결과를 패널에 반영한다 |
| F05-R2 | WHILE | **WHILE** 추론 진행 중, 시스템은 직전 확률을 유지 표시하고 갱신 인디케이터만 추가한다 |
| F05-R3 | Ubiquitous | 시스템은 추론·시뮬레이션 전 과정에서 외부 네트워크 요청을 0건으로 유지한다 |
| F05-R4 | IF-THEN | **IF** wasm 로드가 실패하면, **THEN** 시스템은 순수 JS 통계 폴백으로 전환하고 예측 패널을 "간이 추정" 배지와 함께 유지한다(빈 패널 금지) |
| F05-R5 | IF-THEN | **IF** 모델 파일 다운로드가 미완료인 상태에서 조작이 발생하면, **THEN** 시스템은 조작을 허용하되 "모델 준비 중" 표시 후 로드 완료 시 자동으로 최신 상태를 추론한다 |
| F05-R6 | IF-THEN | **IF** Worker가 응답하지 않으면(타임아웃), **THEN** 시스템은 Worker를 재기동하고 1회 재시도, 재실패 시 통계 폴백으로 전환한다 |

## 6. 시나리오·예외

- **Happy Path**: 슬라이더 조작 → Worker 추론 → 패널 확률 갱신 → F07 시각화 전달
- **Edge Cases** (물음표 → 해소 R):
  - 구형 브라우저에서 wasm이 안 뜨면? → JS 폴백 + 배지, 기능 유지 (F05-R4)
  - 첫 진입 직후 모델 로딩 전에 만지면? → 조작 허용 + 로드 후 자동 추론 (F05-R5)
  - Worker가 죽으면(메모리 부족 등)? → 재기동 1회 → 폴백 (F05-R6)
  - 심사자가 Network 탭으로 검증하면? → 외부 요청 0건 (F05-R3)

## 7. 수용 기준 (Given-When-Then)

- **Given** DevTools Network 오픈 **When** 전술 조작·추론 반복 **Then** 외부 API 요청이 0건이다
- **Given** 재방문(오프라인) **When** S1 진입·조작 **Then** 추론이 동작한다(Service Worker 캐시)
- **Given** wasm 차단 환경(강제 실패) **When** 조작 **Then** "간이 추정" 배지와 함께 확률이 표시된다
- **Given** 추론 반복 중 **When** D&D·슬라이더 조작 **Then** 메인 스레드 인터랙션이 끊기지 않는다

## 8. 데이터·상태·비기능

- **데이터**: `public/model/*.onnx` + `ort.wasm` — Vercel 직접 서빙 (P4)
- **상태**: `prediction{home, draw, away}` + `engineStatus(ready|loading|fallback)` — Worker 메시지: `{type: infer|cancel, payload}` `[설계 결정]`
- **비기능**: 단일 추론 ≤50ms `[추정 설계목표, 실측 필요]` (P4) · 외부 요청 0건 · 학습층 실명 데이터는 모델 가중치로만 존재, 클라이언트 JSON에 실명 미포함 (P7·P10 레이어 분리)
