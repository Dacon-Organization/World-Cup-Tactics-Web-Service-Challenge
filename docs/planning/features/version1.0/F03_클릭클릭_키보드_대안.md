# F03. 클릭-클릭 대안 + 키보드 경로

> **우선순위**: P0 · **담당 화면**: S1 · **근거**: P5(WCAG 2.2 SC 2.5.7 AA) · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "드래그가 어려운 사용자도 탭 두 번(또는 키보드만)으로 선수를 옮길 수 있다."
- **Why this priority**: WCAG 2.5.7(AA)은 모든 드래그에 단일 포인터 대안을 요구 — F02의 접근성 필수 짝. 심사자가 데스크톱 키보드로 시연할 가능성도 방어.
- **Independent Test**: 마우스 드래그를 한 번도 쓰지 않고 선수 이동을 완료하는 것으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: F02와 동일 (피치·토큰 렌더 완료)
- **Post-condition**: F02와 동일 (좌표 커밋 → 예측 트리거) — **두 경로의 결과 상태는 완전 동일**

## 3. UI/UX·네비게이션

- 클릭-클릭: 토큰 탭 → 선택 하이라이트(테두리+확대) → 목적지(슬롯 또는 피치 지점) 탭 → 이동. 다른 토큰 탭 시 선택 전환
- 키보드: Tab(토큰 포커스 순회) → Space(선택) → 방향키(위치 이동, 1회당 피치 2% 단위 `[설계 결정]`) → Enter(확정) → Esc(취소) — 2.5.7과 독립 요건 (P5)
- 포커스 링은 브라우저 기본보다 강화된 시각 표시(피치 배경 위 대비 확보)

## 4. 상세 로직·설계 근거

- 선택 상태는 단일(한 번에 한 토큰) — F02-R3 단일 드래그 원칙과 대칭
- 클릭-클릭·키보드 경로도 드롭존 스냅 규칙(F02-R2) 동일 적용 — 조작 수단만 다르고 규칙은 하나
- **설계 근거**: SC 2.5.7 Dragging Movements(AA)와 키보드 접근성은 독립 요건 (P5)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F03-R1 | WHEN | **WHEN** 토큰을 탭하면, 시스템은 해당 토큰을 선택 상태로 표시한다 |
| F03-R2 | WHEN | **WHEN** 선택 상태에서 목적지를 탭하면, 시스템은 토큰을 그 위치로 이동시키고 선택을 해제한다 |
| F03-R3 | WHEN | **WHEN** 키보드 포커스 토큰에서 Space가 눌리면, 시스템은 선택 모드로 진입하고 방향키 이동을 활성화한다 |
| F03-R4 | IF-THEN | **IF** 선택 상태에서 다른 토큰을 탭하면, **THEN** 시스템은 이동이 아니라 선택 전환으로 처리한다 |
| F03-R5 | IF-THEN | **IF** 선택 상태에서 Esc(또는 피치 밖 탭)가 입력되면, **THEN** 시스템은 선택을 해제하고 위치를 변경하지 않는다 |
| F03-R6 | IF-THEN | **IF** 방향키 이동으로 피치 경계를 넘으려 하면, **THEN** 시스템은 경계에서 이동을 멈춘다(피치 밖 좌표 불가) |

## 6. 시나리오·예외

- **Happy Path (클릭-클릭)**: 토큰 탭 → 하이라이트 → 빈 지점 탭 → 이동 완료 → 예측 갱신
- **Happy Path (키보드)**: Tab ×n → Space → ↑↑→ → Enter → 이동 완료
- **Edge Cases** (물음표 → 해소 R):
  - 옮기려다 다른 선수를 잘못 탭하면? → 선택 전환일 뿐 이동 아님 (F03-R4)
  - 선택해 놓고 마음이 바뀌면? → Esc/밖 탭으로 무해 해제 (F03-R5)
  - 방향키를 계속 누르면 선수가 화면 밖으로? → 경계 정지 (F03-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 마우스 사용 불가 가정 **When** 탭 2회로 토큰 이동 **Then** F02 드래그와 동일한 결과 상태가 된다
- **Given** 키보드만 사용 **When** Tab→Space→방향키→Enter **Then** 이동이 완료되고 포커스 표시가 전 과정에서 보인다
- **Given** 토큰 선택 상태 **When** 다른 토큰 탭 **Then** 이동 없이 선택만 전환된다

## 8. 데이터·상태·비기능

- **상태**: F02와 동일 스토어 경로(`positions[11]` 커밋) — 별도 상태 없음, `selection`은 로컬
- **비기능**: WCAG 2.2 SC 2.5.7 AA 충족 근거 문서화(기획서 8절 인용) · 포커스 대비 비율 확보 · 스크린리더 라벨("~ 선택됨, 방향키로 이동")
