# F06. 몬테카를로 시뮬레이션

> **우선순위**: P0 · **담당 화면**: S1(S2b·S3 재사용) · **근거**: P12·P5 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 자신의 전술로 경기를 수천 번 돌려본 분포(확률+신뢰구간)를 받는다."
- **Why this priority**: 단일 추론을 "시뮬레이션 경험"으로 확장 — 확률 밴드(F07)의 데이터 원천이자 538·Opta와 같은 방법론임을 보이는 기술 인정 지점.
- **Independent Test**: 빠른 모드 실행 → 반복 횟수·밴드 표시, 정밀 모드 실행 → 진행률·취소 동작만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: F05 추론 가능 상태(정상 또는 폴백)
- **Post-condition**: 확률 + Wilson 신뢰구간이 산출되어 F07에 전달됨

## 3. UI/UX·네비게이션

- 빠른 모드(기본): 조작 후 자동 실행, 스피너만
- 정밀 모드(옵션): 접힌 세부 설정에서 명시적 실행 — **진행률 바 + 취소 버튼 필수** (P5 10초 임계)
- 결과에 반복 횟수 명시("5,000회 시뮬레이션") — 정직 고지

## 4. 상세 로직·설계 근거

- 반복 횟수: **빠른 5,000 / 정밀 25,000** (P12 — 538 클럽 20,000·Opta 25,000 정합, ±1%p 밴드에 N≥10,000)
- Wilson score 신뢰구간(순수 JS, 즉시 계산) — 0%/100% 근접에서도 [0,1] 내부 (P12)
- Web Worker 실행, 청크 단위 진행 보고(예: 500회마다) `[설계 결정]` — 취소 즉시성 확보
- **설계 근거**: P3의 1,000/10,000 제안을 P12가 공개 사례 정합으로 상향(원장 기록)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F06-R1 | WHEN | **WHEN** 전술 커밋 debounce가 끝나면, 시스템은 빠른 모드(5,000회)를 자동 실행한다 |
| F06-R2 | WHEN | **WHEN** 정밀 모드가 실행되면, 시스템은 진행률을 표시하고 취소 버튼을 노출한다 |
| F06-R3 | WHEN | **WHEN** 시뮬레이션이 완료되면, 시스템은 확률·Wilson 밴드·반복 횟수를 함께 표시한다 |
| F06-R4 | IF-THEN | **IF** 취소 버튼이 눌리면, **THEN** 시스템은 진행 중 청크 경계에서 즉시 중단하고 직전 완료 결과를 유지한다 |
| F06-R5 | IF-THEN | **IF** 정밀 모드가 10초를 초과하면, **THEN** 시스템은 진행률·취소를 유지한 채 계속하며 완료 후 소요 시간을 함께 표기한다 |
| F06-R6 | IF-THEN | **IF** 시뮬레이션 중 전술이 변경되면, **THEN** 시스템은 진행 중 계산을 취소하고 최신 상태로 빠른 모드를 재실행한다(F04-R4와 동일 원칙) |
| F06-R7 | IF-THEN | **IF** 저성능 환경에서 빠른 모드가 3초를 초과하면, **THEN** 시스템은 다음 자동 실행부터 반복 횟수를 절반으로 낮추고 그 사실을 반복 횟수 표기에 반영한다 `[설계 결정]` |

## 6. 시나리오·예외

- **Happy Path**: 조작 → 빠른 모드 자동 실행(≤1초 목표) → 확률+밴드 갱신 → (선택) 정밀 모드 → 진행률 → 완료
- **Edge Cases** (물음표 → 해소 R):
  - 정밀 모드 걸어놓고 지루해지면? → 취소 즉시 반영, 직전 결과 보존 (F06-R4)
  - 계산 중 전술을 또 바꾸면? → 취소 후 최신 재실행 (F06-R6)
  - 오래된 폰에서 빠른 모드도 느리면? → 반복 수 자동 강등 + 정직 표기 (F06-R7)

## 7. 수용 기준 (Given-When-Then)

- **Given** 기본 상태 **When** 슬라이더 조작 후 대기 **Then** 자동으로 빠른 모드가 실행되고 반복 횟수가 표시된다
- **Given** 정밀 모드 실행 중 **When** 취소 클릭 **Then** 1초 내 중단되고 직전 결과가 유지된다
- **Given** 정밀 모드 완료 **Then** 밴드 폭이 빠른 모드보다 좁다(N 증가 → SE 감소 관찰 가능)

## 8. 데이터·상태·비기능

- **상태**: `simulation{status, progress, iterations, result{p, wilson}}` — Worker 메시지에 `cancel` 포함(F05와 공유 프로토콜)
- **비기능**: 빠른 ≤1초 목표·정밀 ≤10초/초과 시 진행률+취소 (P5·P12) · Wilson 계산은 폴백 모드에서도 동작(순수 JS, P12)
