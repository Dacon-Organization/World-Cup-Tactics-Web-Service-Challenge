# F14. 동적 OG 이미지

> **우선순위**: P2 · **담당 화면**: 메타 산출물(화면 아님) · **근거**: P8·P7 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "카톡에 링크를 붙여넣으면, 내 전술 카드가 미리보기로 뜬다 — 받는 사람이 앱을 열기 전부터."
- **Why this priority**: 카카오톡 미리보기가 한국 유통의 관문이며, **OG 스크래핑만으로 성립(앱 키 불필요)** — "심사자 별도 키 없이" 규칙과 정합 (P8).
- **Independent Test**: 공유 URL을 카카오 공유 디버거(또는 OG 미리보기 도구)에 넣어 카드 이미지가 뜨는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: 상태 URL(F13) 존재. Vercel 배포 환경(서버 렌더 필요 — OG는 유일한 서버 산출물)
- **Post-condition**: 1200×630 카드 이미지가 OG 태그로 응답됨

## 3. UI/UX·네비게이션

- 산출물: 1200×630 PNG — 내 전술 카드(F16)의 시각 요소를 서버 렌더로 재현
- 표기: 가공명·자체 SVG·flag-icons만 — **P7 에셋 가이드 전면 적용**

## 4. 상세 로직·설계 근거

- Next.js `opengraph-image.tsx` + `ImageResponse` — satori 내장, 직접 의존 불필요 (P8)
- URL의 state 파라미터를 파싱해 카드 구성 — F13과 동일 파서 공유(단일 스키마 원칙)
- 카톡 스크랩은 80/443 포트 + `og:title`/`og:image` 기반, **스크랩 캐시 존재** → 시연 전 카카오 공유 디버거로 캐시 갱신(체크리스트 등재) (P8)
- OG 이미지에 폰트 사용 시: 산출물은 PNG이므로 폰트 재배포 아님 → OFL 문제 없음 (P7·P8)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F14-R1 | WHEN | **WHEN** 상태 URL이 스크랩되면, 시스템은 해당 전술의 1200×630 카드 이미지를 OG 태그로 응답한다 |
| F14-R2 | Ubiquitous | 시스템은 OG 이미지에 실명·이니셜·연상 표기와 FIFA IP 요소를 절대 포함하지 않는다 |
| F14-R3 | IF-THEN | **IF** state 파싱이 실패하면, **THEN** 시스템은 기본 브랜드 카드(서비스명+기본 비주얼)를 응답한다(깨진 미리보기 금지) |
| F14-R4 | IF-THEN | **IF** 이미지 생성이 실패하면(서버 오류), **THEN** 시스템은 정적 기본 OG 이미지로 폴백한다 |
| F14-R5 | IF-THEN | **IF** state 없는 루트 URL이 스크랩되면, **THEN** 시스템은 서비스 소개용 기본 카드를 응답한다 |

## 6. 시나리오·예외

- **Happy Path**: 카톡에 URL 붙여넣기 → 스크래퍼 요청 → 전술 카드 응답 → 미리보기 표시
- **Edge Cases** (물음표 → 해소 R):
  - 변조된 state로 스크랩되면? → 기본 브랜드 카드 (F14-R3)
  - satori 렌더가 서버에서 죽으면? → 정적 폴백 (F14-R4)
  - 루트 URL만 공유하면? → 소개 카드 (F14-R5)
  - 카드를 수정했는데 카톡 미리보기가 그대로면? → 스크랩 캐시 — 디버거로 갱신(운영 체크리스트, 기능 결함 아님)

## 7. 수용 기준 (Given-When-Then)

- **Given** 상태 URL **When** OG 미리보기 도구로 확인 **Then** 전술 카드 이미지가 1200×630으로 응답된다
- **Given** 카카오톡 **When** URL 전송 **Then** 앱 키·SDK 없이 미리보기 카드가 뜬다
- **Given** 생성된 모든 카드 **Then** 실명·FIFA IP 요소가 0건이다
