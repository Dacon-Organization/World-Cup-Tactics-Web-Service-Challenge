# F04. 전술 슬라이더 4종

> **우선순위**: P0 · **담당 화면**: S1 · **근거**: P5·ADR-002 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 감독의 언어(라인 높이·압박 강도·공격 폭·템포)로 팀 성향을 미세 조정한다."
- **Why this priority**: "이분법 토글보다 연속 슬라이더가 감독 행위감을 살린다"(P5) — 감동 경험 25점의 두 번째 축이자 모델 입력 피처.
- **Independent Test**: 슬라이더 1개를 움직여 100ms 내 피치 프리뷰 변화를 보이는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: S1 기본 상태(ADR-004: 4종 전부 중앙값)
- **Post-condition**: 슬라이더 값 커밋 → 시각 프리뷰 반영 + debounce 후 시뮬레이션(F06) 트리거

## 3. UI/UX·네비게이션

- 구성: 라인 높이 · 압박 강도 · 공격 폭 · 템포 (ADR-002)
- 라벨: 감독 용어 + "낮음/보통/높음" 질적 라벨 병기 (P5)
- 연속값(0~100) `[설계 결정]`, 모바일은 하단 시트, 데스크톱은 우측 패널 상단 (화면흐름 S1)
- 시각 프리뷰: 라인 높이→수비 라인 가이드 위치, 압박 강도→압박 영역 음영, 공격 폭→좌우 산개 가이드, 템포→전진 화살표 밀도 `[설계 결정]`

## 4. 상세 로직·설계 근거

- 조작 즉시(≤100ms) 시각 프리뷰 — 추론과 분리된 경량 렌더 (P5 100ms 직접 조작감)
- 값 변경은 debounce(조작 종료 후) 시뮬레이션 트리거 — 드래그 중 추론 폭주 방지 (P12 상호작용 흐름)
- 4종 값은 모델 입력 피처로 매핑 (ML 설계 단계 구체화, ADR-002 Consequences 참조)
- **설계 근거**: 슬라이더 채택·라벨 병기는 P5, 4종 구성은 ADR-002

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F04-R1 | WHEN | **WHEN** 슬라이더 값이 변하면, 시스템은 100ms 내 해당 시각 프리뷰를 피치에 반영한다 |
| F04-R2 | WHEN | **WHEN** 슬라이더 조작이 끝나면(debounce), 시스템은 시뮬레이션(F06)을 자동 실행한다 |
| F04-R3 | WHILE | **WHILE** 시뮬레이션 진행 중, 시스템은 추가 슬라이더 조작을 허용하고 완료 시 최신 값으로 재실행한다(마지막 값 승리) |
| F04-R4 | IF-THEN | **IF** 시뮬레이션 진행 중 값이 또 변경되면, **THEN** 시스템은 진행 중 계산을 취소하고 최신 값으로 재시작한다(구식 결과 표시 금지) |
| F04-R5 | IF-THEN | **IF** 프리뷰 렌더가 100ms를 초과하는 저성능 환경이면, **THEN** 시스템은 프리뷰 효과를 단계적으로 축소(음영→선만)하되 값 반영은 유지한다 |
| F04-R6 | IF-THEN | **IF** 터치 조작이 슬라이더 트랙을 벗어나며 끝나면, **THEN** 시스템은 마지막 유효 값으로 확정한다(값 유실 금지) |

## 6. 시나리오·예외

- **Happy Path**: 압박 강도 올림 → 음영 즉시 변화 → 손 뗌 → 1초 내 확률 갱신
- **Edge Cases** (물음표 → 해소 R):
  - 슬라이더를 빠르게 왔다갔다하면 옛날 계산 결과가 나중에 도착해 덮어쓰나? → 취소 후 최신 재시작 (F04-R4)
  - 계산 중에 또 만지면 잠기나? → 잠금 없음, 마지막 값 승리 (F04-R3)
  - 저사양 폰에서 프리뷰가 버벅이면? → 효과 축소, 값은 보존 (F04-R5)

## 7. 수용 기준 (Given-When-Then)

- **Given** 기본 상태 **When** 슬라이더 1개 조작 **Then** 100ms 내 피치 시각 변화가 관찰된다
- **Given** 임의 상태 **When** 4종 각각 조작 **Then** 각각 예측 확률(F05 표시값)이 변한다
- **Given** 시뮬레이션 진행 중 **When** 값 재변경 **Then** 최종 표시 결과는 항상 마지막 값 기준이다
- **Given** 모바일 **When** 슬라이더 조작 **Then** 트랙·핸들이 터치 타겟 ≥1cm이다

## 8. 데이터·상태·비기능

- **상태(Zustand)**: `tactics{lineHeight, pressing, width, tempo}` — F13 URL 인코딩 대상
- **연계**: 값 커밋 → debounce → F06. Worker 취소 메시지 프로토콜은 F06와 공유
- **비기능**: 프리뷰 ≤100ms (P5) · 질적 라벨 스크린리더 노출 · 터치 타겟 ≥1cm (P5)
