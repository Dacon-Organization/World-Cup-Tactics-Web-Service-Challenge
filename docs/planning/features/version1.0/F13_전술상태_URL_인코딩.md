# F13. 전술 상태 URL 인코딩

> **우선순위**: P2 · **담당 화면**: S3(생성은 S1·S2b) · **근거**: P8 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자가 공유한 URL을 열면, 받은 사람의 브라우저에서 같은 전술이 복원되고 확률이 다시 계산된다."
- **Why this priority**: 공유 3계층의 뿌리 — URL이 곧 저장 수단(ADR-001: 서버 저장 없음). 수신자 재추론이라 공유에 서버가 불필요 (P8).
- **Independent Test**: URL 복사 → 시크릿 창에서 열기 → 동일 전술 복원 + 자동 재추론만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: 공유할 전술 상태 존재(S1) 또는 개입 시나리오 존재(S2b)
- **Post-condition**: 생성 — 상태가 담긴 URL / 복원 — S3에서 상태 복원+재추론 완료

## 3. UI/UX·네비게이션

- 생성: 공유 버튼(F15) 내부 동작 — 사용자에게는 URL 문자열로만 노출
- 복원: `/t/[state]` 진입 → 복원 화면(S3) → "내 전술로 바꿔보기" CTA → S1 전환(상태 이어받기)

## 4. 상세 로직·설계 근거

- 인코딩 대상: 포메이션 ID + 11명 정규화 좌표(양자화) + 슬라이더 4값 + (리플레이) matchId·개입 분
- 포맷: 버전 프리픽스 포함(`v1.`) base64url `[설계 결정]` — 향후 스키마 변경 대비
- 좌표 양자화(예: 0~100 정수)로 길이 절약 `[설계 결정]` — **URL 길이가 카톡·SNS에서 잘리지 않는 한계선은 P17 실측 대기**
- **설계 근거**: "공유 URL은 계산 결과가 아닌 전술 상태를 인코딩 → 수신자 브라우저에서 재추론" (P8)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F13-R1 | WHEN | **WHEN** 공유가 실행되면, 시스템은 현재 상태를 버전 프리픽스 포함 문자열로 인코딩한 URL을 생성한다 |
| F13-R2 | WHEN | **WHEN** 유효한 상태 URL로 진입하면, 시스템은 상태를 복원하고 수신자 브라우저에서 자동 재추론한다 |
| F13-R3 | IF-THEN | **IF** state 파싱이 실패하면(손상·임의 변조), **THEN** 시스템은 오류 화면 없이 기본 상태(F08)의 S1로 안내하고 "링크를 읽을 수 없어 기본 전술로 시작합니다" 토스트를 표시한다 |
| F13-R4 | IF-THEN | **IF** state의 버전 프리픽스가 미지원 버전이면, **THEN** 시스템은 F13-R3과 동일하게 처리한다(구버전 링크 크래시 금지) |
| F13-R5 | IF-THEN | **IF** state 값이 스키마 범위를 벗어나면(좌표>100, 슬라이더>최대 등), **THEN** 시스템은 해당 값을 경계값으로 클램프해 복원한다(부분 손상 관용) |
| F13-R6 | IF-THEN | **IF** state에 존재하지 않는 matchId가 포함되면, **THEN** 시스템은 리플레이 컨텍스트를 버리고 전술만 S1으로 복원한다 |

## 6. 시나리오·예외

- **Happy Path**: S1에서 공유 → URL 생성 → 수신자 열기 → S3 복원·재추론 → "내 전술로 바꿔보기"
- **Edge Cases** (물음표 → 해소 R):
  - 누가 URL을 손으로 고쳐서 이상한 값을 넣으면? → 파싱 실패는 기본 상태 안내, 범위 초과는 클램프 (F13-R3·R5)
  - 다음 버전에서 스키마가 바뀌면 옛 링크는? → 버전 프리픽스로 감지, 안전 폴백 (F13-R4)
  - 조작된 matchId로 리플레이를 열려 하면? → 전술만 복원 (F13-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 임의 전술 **When** URL 생성→새 브라우저에서 열기 **Then** 포메이션·좌표·슬라이더가 동일하게 복원된다
- **Given** 복원 화면 **Then** 재추론이 자동 실행되고 외부 API 호출은 0건이다(F05-R3 승계)
- **Given** state를 임의 변조한 URL **When** 진입 **Then** 크래시·백지 없이 기본 상태 또는 클램프 복원이 된다

## 8. 데이터·상태·비기능

- **URL 스키마**: `/t/v1.<base64url>` `[설계 결정]` — 상세 필드 배치는 데이터 설계에서 확정
- **보안 경계**: state는 표현 데이터일 뿐 실행 코드 아님 — 파싱은 화이트리스트 스키마 검증만(eval류 금지)
- **비기능**: URL 길이 한계 P17 실측 대기(비블로킹) · 개인 정보 미포함(전술 값만)
