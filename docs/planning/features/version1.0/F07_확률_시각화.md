# F07. 확률 시각화

> **우선순위**: P0 · **담당 화면**: S1(S2b·S3 재사용) · **근거**: P12·P5 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 '62%'라는 숫자 하나가 아니라, 100번 중 62번이 어떤 모습인지 본다."
- **Why this priority**: 단일 수치 오독 편향 방지(P12) — 확률을 정직하게 전달하는 것이 서비스 신뢰도의 기반이자 감동 경험의 마감 품질.
- **Independent Test**: 동일 확률에 대해 아이콘 배열·퍼센트·밴드가 함께 표시되고 HOPs 토글이 동작하는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: F06 결과(확률+Wilson 밴드) 수신
- **Post-condition**: 3중 표현(배열+퍼센트+밴드) 렌더 완료, 공유 카드(F16)·이모지 그리드(F15)의 시각 원형 제공

## 3. UI/UX·네비게이션

전부 P12 확정:
- **10×10 아이콘 배열**: top/row 채움 방식, central/random 편향 배치 회피, 퍼센트 숫자 병기
- **Wilson 밴드**: 확률 막대에 신뢰구간 표시(빠른/정밀 모드에 따라 폭 변화)
- **HOPs 옵션**: 400ms/프레임 재생·정지 토글 — 사전 생성 프레임(각 프레임=몬테카를로 1회), <100ms 금지
- 스크린리더: "승리 확률 62%, 100번 중 62번" 텍스트 라벨

## 4. 상세 로직·설계 근거

- HOPs 프레임은 F06 결과에서 사전 렌더 — 추론과 분리 (P12)
- 아이콘 배열은 순수 SVG/CSS — 시각화 라이브러리 의존 없음 `[설계 결정]` (번들 예산)
- **설계 근거**: 역할 분담(단일 확률→배열, 시나리오 대조→HOPs)은 P12 채택 결론

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F07-R1 | WHEN | **WHEN** 시뮬레이션 결과가 도착하면, 시스템은 배열·퍼센트·밴드 3중 표현을 동기 갱신한다 |
| F07-R2 | WHERE | **WHERE** HOPs 옵션이 켜져 있으면, 시스템은 400ms/프레임으로 사전 생성 프레임을 재생한다 |
| F07-R3 | Ubiquitous | 시스템은 확률을 항상 퍼센트 숫자와 병기한다(그래픽 단독 표시 금지) |
| F07-R4 | IF-THEN | **IF** 확률이 0% 또는 100%에 근접하면, **THEN** 시스템은 Wilson 밴드를 사용해 [0,1] 범위를 벗어나는 표시를 하지 않는다 |
| F07-R5 | IF-THEN | **IF** HOPs 재생 중 새 결과가 도착하면, **THEN** 시스템은 재생을 중지하고 새 프레임 세트로 교체한 뒤 정지 상태에서 시작한다 |
| F07-R6 | IF-THEN | **IF** 사용자가 OS 수준 동작 축소(prefers-reduced-motion)를 설정했으면, **THEN** 시스템은 HOPs 자동 재생을 비활성화하고 정적 배열만 표시한다 |

## 6. 시나리오·예외

- **Happy Path**: 결과 도착 → 배열 62칸 채움 + "62%" + 밴드 → HOPs 토글 → 프레임 재생
- **Edge Cases** (물음표 → 해소 R):
  - 확률이 99.9%면 밴드가 100%를 뚫나? → Wilson이라 불가능 (F07-R4)
  - HOPs 재생 중 전술을 바꾸면 옛 프레임이 돌아가나? → 중지·교체 (F07-R5)
  - 멀미 민감 사용자는? → reduced-motion 존중 (F07-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 임의 결과 **When** 패널 확인 **Then** 배열·퍼센트·밴드가 동일 확률을 가리킨다(상호 불일치 0)
- **Given** HOPs 켜짐 **When** 재생 **Then** 프레임 간격이 400ms±오차이며 정지 토글이 즉시 동작한다
- **Given** 스크린리더 **When** 패널 포커스 **Then** 확률이 텍스트로 읽힌다

## 8. 데이터·상태·비기능

- **상태**: `visualization{mode, hopsPlaying, frames[]}` — frames는 세션 메모리만(URL 비인코딩)
- **연계**: 배열 포맷은 F15 이모지 그리드·F16 카드의 시각 원형(동일 데이터 3표현)
- **비기능**: HOPs 400ms/프레임·<100ms 금지 (P12) · prefers-reduced-motion 존중 · 라이브러리 무의존
