# F02. 선수 드래그앤드롭

> **우선순위**: P0 · **담당 화면**: S1 · **근거**: P5(NN/g 1차 출처) · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 선수 토큰을 손가락(마우스)으로 집어 피치 위 원하는 위치로 옮긴다."
- **Why this priority**: 대회 핵심 요건("직접 드래그, 클릭, 배치, 조작") 그 자체 — 감동 경험 25점의 1차 접점.
- **Independent Test**: 토큰 1개를 드래그해 옮기고 Esc로 취소하는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: S1에 피치·토큰 렌더 완료(F01). 프리셋 전환 애니메이션 비진행(F01-R2).
- **Post-condition**: 토큰이 새 좌표에 고정, 상태 갱신 → 예측 파이프라인 트리거(F05).

## 3. UI/UX·네비게이션

전부 P5(NN/g) 확정 스펙:
- 아이들: grab-handle 어포던스, hover 시 `grab` 커서
- 드래그 중: 반투명 고스트(원위치 잔상) + 드롭섀도("들려 있음"), 모바일은 손가락 위 오프셋 표시
- 드롭존: 시각 경계보다 넓은 스냅 마그네티즘
- 드롭/취소: 완료 ~100ms 애니메이션, 피치 밖 mouseup = 원위치 복귀, Esc = 취소

## 4. 상세 로직·설계 근거

- 좌표는 정규화(0~1) 저장 — 뷰포트 무관 (F01과 공유)
- 드래그 종료 시에만 상태 커밋 — 드래그 중 프레임마다 스토어 갱신 금지(리렌더 폭주 방지) `[설계 결정]`
- 상태 커밋 → debounce 후 시뮬레이션(F06) — 즉시 프리뷰는 F04 경로와 동일 규칙
- **설계 근거**: 드래그 패턴 전체가 NN/g 1차 출처 (P5)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F02-R1 | WHEN | **WHEN** 토큰에서 드래그가 시작되면, 시스템은 고스트·드롭섀도·커서 변화를 즉시 표시한다 |
| F02-R2 | WHEN | **WHEN** 드롭존 근접 상태에서 드롭하면, 시스템은 토큰을 슬롯에 스냅시키고 ~100ms 애니메이션으로 고정한다 |
| F02-R3 | WHILE | **WHILE** 드래그 진행 중, 시스템은 다른 토큰의 드래그 시작을 무시한다(단일 드래그 원칙) |
| F02-R4 | IF-THEN | **IF** 드래그 중 Esc가 눌리면, **THEN** 시스템은 드래그를 취소하고 토큰을 원위치로 복귀시킨다 |
| F02-R5 | IF-THEN | **IF** 피치 밖에서 드롭(또는 포인터 이탈)하면, **THEN** 시스템은 원위치 복귀시키고 상태를 변경하지 않는다 |
| F02-R6 | IF-THEN | **IF** 드래그 중 화면 회전·리사이즈가 발생하면, **THEN** 시스템은 진행 중 드래그를 취소(원위치 복귀)하고 새 뷰포트로 재렌더링한다 |
| F02-R7 | IF-THEN | **IF** 멀티터치로 두 번째 손가락이 다른 토큰에 닿으면, **THEN** 시스템은 두 번째 터치를 무시한다(F02-R3의 터치 구체화) |

## 6. 시나리오·예외

- **Happy Path**: 토큰 터치 → 고스트 표시 → 이동 → 슬롯 근접 드롭 → 스냅 고정 → 예측 갱신(F05)
- **Edge Cases** (물음표 → 해소 R):
  - 드래그하다 마음이 바뀌면? → Esc 또는 피치 밖 드롭으로 취소 (F02-R4·R5)
  - 두 손가락으로 두 토큰을 동시에 잡으면? → 첫 토큰만 유효 (F02-R7)
  - 드래그 중 전화가 오거나 화면이 돌아가면? → 안전하게 취소·복귀 (F02-R6)
  - 스크롤하려다 토큰이 끌려오면? → 토큰 외 영역은 스크롤, 토큰은 드래그 — 터치 시작점으로 구분 `[설계 결정: long-press 지연 도입 여부는 구현 중 실기기 테스트로 판단]`

## 7. 수용 기준 (Given-When-Then)

- **Given** 데스크톱 **When** 마우스로 토큰을 슬롯 근처에 드롭 **Then** 토큰이 슬롯에 스냅되고 100ms 내 고정된다
- **Given** 모바일 실기기 **When** 터치 드래그로 토큰 이동 **Then** 피드백이 손가락에 가리지 않고 이동이 완료된다
- **Given** 드래그 진행 중 **When** Esc 입력 **Then** 토큰이 원위치로 복귀하고 예측은 재실행되지 않는다
- **Given** 드래그 진행 중 **When** 화면 회전 **Then** 크래시·좌표 왜곡 없이 취소·복귀된다

## 8. 데이터·상태·비기능

- **상태(Zustand)**: `positions[11]` 갱신(드롭 시 1회 커밋) · `dragState`(로컬 컴포넌트 상태, 스토어 비저장)
- **연계**: 커밋 → F04와 동일 debounce 경로로 F06 트리거
- **비기능**: 재배치 ~100ms (P5) · 터치 타겟 ≥1cm×1cm (P5) · 클릭-클릭 대안은 F03(WCAG 2.5.7 — 본 기능 단독으로는 접근성 요건 미충족임을 명시)
