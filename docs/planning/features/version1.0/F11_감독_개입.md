# F11. "내가 감독이라면" 개입

> **우선순위**: P1 · **담당 화면**: S2b · **근거**: P11·비하 금지 규칙 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 경기의 한 시점을 골라 전술을 바꿔보고, '다른 선택지'의 확률을 실제 결과와 나란히 본다."
- **Why this priority**: 서비스 이름값("내가 축구 감독이라면")의 완성 — 문제의식(10위/8위 컷)에 대한 인터랙티브 응답. 참신성의 정점.
- **Independent Test**: 시점 선택 → 전술 변경 → 비교 패널 표시의 한 사이클만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: S2b 타임라인 렌더 완료(F10)
- **Post-condition**: 개입 시나리오(시점+변경 전술)의 재예측 결과가 실제 결과와 병렬 표시. F13 공유 대상 상태.

## 3. UI/UX·네비게이션

- 타임라인 노드(또는 임의 분) 선택 → "이 시점에 개입하기" → 전술보드 오버레이(F01~F04 재사용)
- 비교 패널: 좌 "실제 경기"(스코어·이벤트) / 우 "당신의 시나리오"(확률+밴드) — 나란히, 우열 표기 없음
- 결과는 항상 확률+Wilson 밴드 — "이겼을 것" 단정 표기 금지 (정직 고지 원칙)

## 4. 상세 로직·설계 근거

- 개입 = 해당 분 이후의 조건(전술 상태)을 바꾼 재추론 — 이전 이벤트는 사실로 고정
- 재추론·시뮬레이션은 F05·F06 엔진 그대로 재사용(입력에 개입 시점 컨텍스트 추가)
- **카피 규칙(비하 금지, 카피 단위 관리)**: "다른 선택지가 있었다" 관점만. 실제 감독·선수의 판단을 "잘못/실수/패착"으로 규정하는 어휘 전면 금지. 46' 교체 등은 사실 서술만 (P11)

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F11-R1 | WHEN | **WHEN** 개입 시점이 선택되면, 시스템은 전술보드 오버레이를 열고 해당 시점의 실제 배치를 초기값으로 제시한다 |
| F11-R2 | WHEN | **WHEN** 개입 전술이 확정되면, 시스템은 재예측을 실행하고 실제 결과와 병렬 비교 패널을 표시한다 |
| F11-R3 | Ubiquitous | 시스템은 비교 결과를 항상 확률·밴드로 표시하며 단정 표현("이겼다/졌을 것")을 사용하지 않는다 |
| F11-R4 | IF-THEN | **IF** 오버레이에서 변경 없이 닫으면, **THEN** 시스템은 재예측을 실행하지 않고 타임라인 상태로 복귀한다 |
| F11-R5 | IF-THEN | **IF** 재예측 중 다른 시점을 선택하면, **THEN** 시스템은 진행 중 계산을 취소하고 새 시점 흐름을 시작한다(F06-R6 동일 원칙) |
| F11-R6 | IF-THEN | **IF** 경기 종료 분(90'+)을 개입 시점으로 선택하려 하면, **THEN** 시스템은 선택 가능 범위를 마지막 유효 분까지로 제한한다 |

## 6. 시나리오·예외

- **Happy Path**: 남아공전 60' 선택 → 개입 → 슬라이더·배치 변경 → 재예측 → "실제 0-1 / 시나리오 승 42% 무 31% 패 27%" 병렬 표시
- **Edge Cases** (물음표 → 해소 R):
  - 열어만 보고 안 바꾸면? → 계산 없이 복귀 (F11-R4)
  - 계산 중 다른 분을 찍으면? → 취소·재시작 (F11-R5)
  - 종료 휘슬 뒤에 개입? → 범위 제한 (F11-R6)

## 7. 수용 기준 (Given-When-Then)

- **Given** 임의 경기 **When** 시점 선택→전술 변경→확정 **Then** 실제/시나리오가 나란히 표시되고 시나리오 쪽은 확률+밴드다
- **Given** 비교 패널 **Then** 단정·평가 어휘가 0건이다(카피 전수 검수)
- **Given** 개입 확정 상태 **When** 공유(F13) **Then** 경기 ID·개입 시점·전술이 URL에 포함된다

## 8. 데이터·상태·비기능

- **상태**: `intervention{matchId, minute, tactics, positions}` — F13 인코딩 대상
- **연계**: F05·F06 엔진 재사용(신규 추론 경로 없음 — 완성도 리스크 최소화)
- **비기능**: 비교 패널 카피는 조판 단계 비하 금지 전수 검수 목록에 등재 · 재예측 지연 예산은 F06과 동일
