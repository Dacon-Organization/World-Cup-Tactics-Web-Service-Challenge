# F10. 이벤트 타임라인

> **우선순위**: P1 · **담당 화면**: S2b · **근거**: P11 · [인덱스](기능명세_인덱스_v1_0.md)

## 1. 개요 (Overview)

- **User Story**: "사용자는 경기의 흐름(득점·교체)을 분 단위로 따라가며 '그 순간'을 다시 본다."
- **Why this priority**: 개입(F11)의 무대 — 어느 분에 개입할지 고르려면 타임라인이 정확해야 한다. 데이터 정확성이 곧 신뢰.
- **Independent Test**: 타임라인의 전 이벤트가 P11 확정 목록과 1:1 일치하는 것만으로 단독 시연 가능.

## 2. 조건 및 상태 (Conditions)

- **Pre-condition**: S2b 진입, `matchId` 유효(F09-R3 통과)
- **Post-condition**: 이벤트 렌더 완료, 각 이벤트 지점이 F11 개입점·F12 마커로 활성

## 3. UI/UX·네비게이션

- 세로(모바일)/가로(데스크톱) 타임라인: 분 단위 이벤트 노드(득점·교체 IN/OUT)
- xG 표시부: **"Opta xG" 출처 라벨 상시 병기** — 남아공전 1.16/0.90 정밀값 (P11)
- 이벤트 서술은 **사실만**: "46' 교체 투입(FW-A ↔ MF-C)" — 평가 서술 금지 (비하 금지)

## 4. 상세 로직·설계 근거

- 이벤트 원천: **P11 확정 이벤트 목록만** (체코전 67'·80' 득점, 남아공전 63' 실점, 46' 교체 등) — `matches.json`
- **남아공전 경고 이벤트는 P13(FIFA 집계 1 vs 위키 개별 2 상충) 확정 전 미표시** (P11) — 데이터에 넣지 않는 것으로 구현 차단
- xG 모델 차이(FIFA/Opta/FotMob)는 상충이 아니라 모델 차이 → 출처 라벨 병기 원칙 (P11)
- 선수 표기는 가공명(문서 단계에서는 추상 표기) — P7 노출 표면 규칙

## 5. 요구사항 (EARS)

| ID | 패턴 | 요구사항 |
|---|---|---|
| F10-R1 | WHEN | **WHEN** S2b가 로드되면, 시스템은 해당 경기의 확정 이벤트 전체를 분 순서로 렌더한다 |
| F10-R2 | Ubiquitous | 시스템은 xG 수치를 표시할 때 항상 "Opta xG" 출처 라벨을 병기한다 |
| F10-R3 | Ubiquitous | 시스템은 P13 미확정 이벤트(남아공전 경고)를 어떤 화면에도 표시하지 않는다 |
| F10-R4 | IF-THEN | **IF** 이벤트 데이터에 스키마 위반(음수 분·중복 ID 등)이 있으면, **THEN** 시스템은 해당 이벤트만 제외하고 나머지를 렌더하며 콘솔 경고를 남긴다(전체 백지 금지) |
| F10-R5 | IF-THEN | **IF** 타임라인이 뷰포트보다 길면, **THEN** 시스템은 타임라인 영역만 내부 스크롤하고 페이지 가로 스크롤을 만들지 않는다 |
| F10-R6 | IF-THEN | **IF** 이벤트 노드를 연속으로 빠르게 탭하면, **THEN** 시스템은 마지막 탭만 상세 표시한다(중복 팝업 금지) |

## 6. 시나리오·예외

- **Happy Path**: S2b 진입 → 타임라인 렌더 → 67' 득점 노드 탭 → 상세(xG·Opta 라벨) 표시
- **Edge Cases** (물음표 → 해소 R):
  - 데이터 오타로 이상한 이벤트가 있으면? → 개별 제외+경고, 전체 유지 (F10-R4)
  - 남아공전 경고는 왜 없나? → 의도된 미표시(P13 대기) — 기획서에도 명기 (F10-R3)
  - 모바일에서 타임라인이 길면? → 내부 스크롤 (F10-R5)

## 7. 수용 기준 (Given-When-Then)

- **Given** 각 경기 S2b **Then** 표시 이벤트가 P11 확정 목록과 1:1 일치한다(추가·누락 0)
- **Given** xG 표시 지점 **Then** "Opta xG" 라벨이 항상 붙어 있다
- **Given** 남아공전 **Then** 경고 이벤트가 화면에 없다

## 8. 데이터·상태·비기능

- **데이터**: `matches.json.events[]` — `{minute, type(goal|sub), team, actors(가공명 ID), xg?}` `[설계 결정: 상세 스키마는 데이터 설계에서 확정]`
- **비기능**: 이벤트 노드 터치 타겟 ≥1cm · 사실 서술만(비하 금지 카피 검수) · 이벤트 텍스트 스크린리더 host 노출
