## 3. 00장(수집·탐색) 구성안

### 3-1. 수집의 서사 (피드백 ① 전반부)

1. **출처 선정의 이유** — 왜 `jfjelstul/worldcup`인가: P2(후보 감사)→P10(문구 단위 대조)의
   결정 과정을 노트북 안에서 요약 재현 (StatsBomb 재업로드 기각 사유 포함). CC BY-SA 4.0
   고지 → [DATA-LICENSE.md](../../../DATA-LICENSE.md) 링크
2. **수집 방법 상세** — raw.githubusercontent 직접 다운로드 코드 + 커밋 핀 고정(재현성) +
   파일 크기·행수·체크섬 기록 표
3. **캐시 구조** — cookiecutter-data-science 규약(P18 Q5) 채택: `data/raw/`(불변 원본),
   `data/interim/`(마스킹·정제), `data/processed/`(피처 테이블). **raw는 절대 수정 금지** —
   반복 사이클에서 어느 단계 산출물인지 경로가 말해주는 구조
4. **실명 마스킹 셀 이동** — ⚠️ 현행 01장 1-3의 마스킹 셀(NB-R1)을 **00장 로드 직후로
   이동**. 00장이 원본을 만지는 첫 지점이 되므로 경계도 함께 앞당겨야 프로파일링
   출력(값 샘플 노출)에 실명이 새지 않음 (P18 Q1 채택 조건 1과 정합)

### 3-2. 탐색의 상세 (피드백 ① 후반부 — "바로 EDA로 넘어가지 않는다")

EDA(분포·관계의 통계 분석)와 구분되는 **탐색(데이터 실체 파악)** 을 전수 수행:

| 절 | 내용 | 도구 (P18 채택) |
|---|---|---|
| 테이블 전수 조사 | 4개 CSV 각각: 행수·컬럼·dtype·메모리·키 후보 | pandas 직접 |
| 스키마·키 무결성 | PK 유일성, 테이블 간 FK 정합(match_id 등), 고아 레코드 | pandera `DataFrameSchema` |
| 결측 구조 | 컬럼별 결측률 + **결측의 패턴**(특정 연대·대회에 몰리는가) | missingno matrix·bar |
| 프로파일 요약 | 수치·범주 컬럼 통계 일괄 → **커스텀 요약 표**로 압축 표시 | fg-data-profiling `minimal=True` → json 추출 (HTML 통짜 임베드 금지) |
| 값 탐색 | 범주 유니크 값 목록, 이상값 후보(음수 득점 등), 시대별 커버리지 | pandas 직접 |
| 탐색 결론 셀 | "발견한 사실 → 01장 전처리 결정에 넘기는 목록" 마크다운 | — |

**도구는 보조, 서사는 사람** (P18 채택 조건 3): 도구 출력은 출발점 표로 쓰고 각 이상
신호의 해석은 직접 서술. 각 절은 "확인 질문 → 코드 → 답" 구조 유지.

## 4. 반복 사이클 규약 (피드백 ②)

```
탐색(00) → 전처리 결정(01) → EDA(01) → 가공(02) → 모델링(03~07) → 해석(08~09)
   ↑                                                        │
   └────────── 문제 발견 시 해당 상류 단계로 복귀 ──────────┘
```

1. **사이클 정의**: 하류 단계(모델링·해석)에서 발견한 문제로 상류 단계(탐색·전처리·가공)를
   고치는 1회 왕복 = 사이클 1회. 노트북 버전은 `c1`(최초 작성), `c2`, … 로 부른다
2. **기록 이중화**:
   - **각 노트북 서두 "개정 이력" 셀** — 표 형식: `사이클 | 날짜 | 발견(무엇이 문제였나) |
     원인 | 수정 | 영향 범위`. 최초 작성도 c1로 기록
   - **중앙 원장 [notebooks/ITERATION-LOG.md](../../../notebooks/ITERATION-LOG.md)** —
     사이클 단위 상세(발견 경위·대안 비교·기각안 포함). 노트북 셀은 요약, 원장이 정본
     (RESEARCH-LOG와 같은 철학: 발견이 사라지지 않게)
3. **기계 게이트**: 각 장 진입 시 이전 장 산출물을 pandera 스키마로 검증 — 상류 수정이
   하류 계약을 깨면 즉시 명시적 에러 (NB-R5의 도구화, 02장 리키지 assert의 일반화)
4. **노트북_서사 v1.1 반영 예정 조항** (버전업 시 기계적 적용 목록):
   - §2 표에 00장 행 추가, §3에 00장 카드 추가
   - §4 마스킹 경계를 "01장 서두" → "00장 로드 직후"로 개정
   - §5 실행 순서 "01→09" → "00→09"
   - §6에 NB-R7 신설: **WHEN** 사이클 개정이 발생하면 개정 이력 셀과 ITERATION-LOG에
     동시 기록된다 / NB-R8: **IF** 장 진입 pandera 게이트가 실패하면 **THEN** 하류 실행을
     중단하고 원인을 ITERATION-LOG에 기록한다
