# 화면 흐름 v1.0 (상세 설계)

> 기획서 골격 **7·11절의 원고 재료**입니다. 기능 정의는 [기능명세 인덱스 v1.0](../../features/version1.0/기능명세_인덱스_v1_0.md)과
> 개별 기능 문서(F01~F16)를 참조합니다. 본문의 `F1`~`F16` 표기는 `F01`~`F16`과 동일 ID입니다.
> 표기 규약(Pn 역추적·`[설계 결정]`·추상 선수 표기)은 기능명세 인덱스와 동일합니다.

---

## 정보 구조 (IA) — 3페이지 + 1 메타 산출물

```
/                          S1 전술보드 (메인)
├── /replay                S2a 리플레이 경기 선택
│   └── /replay/[matchId]  S2b 리플레이 재생·개입
├── /t/[state]             S3 공유 수신 랜딩
└── opengraph-image        OG 메타 산출물 (화면 아님, F14)
```

| ID | 경로 | 화면 | 담당 기능 | 역할 |
|---|---|---|---|---|
| S1 | `/` | 전술보드 (메인) | F1~F8 | 첫 진입 즉시 기본 전술+예측 표시 (빈 화면 금지) |
| S2a | `/replay` | 경기 선택 | F9 | 3경기 카드 (확정 데이터) |
| S2b | `/replay/[matchId]` | 리플레이 재생·개입 | F10~F12 | 타임라인 → 개입 → 비교 |
| S3 | `/t/[state]` | 공유 수신 랜딩 | F13 | 상태 복원 → 재추론 → "내 전술로 바꿔보기" |

라우팅은 Next.js 15 App Router. S1·S2a는 Server Component 우선, 인터랙션 영역만 Client Component `[설계 결정]`.

## 화면별 구성 (Mobile First — 심사자 모바일 열람 전제)

### S1 — 전술보드 (메인)

**모바일 (기준 설계)**

```
┌─────────────────────┐
│ 헤더: 서비스명·모드 탭 │  ← 리플레이 진입점
├─────────────────────┤
│                     │
│   피치 (자체 SVG)     │  ← F1·F2·F3: 11명 토큰 D&D
│   세로 방향 배치       │     터치 타겟 ≥1cm (P5)
│                     │
├─────────────────────┤
│ 예측 패널 (하단 고정)  │  ← F5·F6·F7: 승/무/패 배열+밴드
├─────────────────────┤
│ 슬라이더 시트 (스와이프) │  ← F4: 4개, 감독 용어 라벨
└─────────────────────┘
```

**데스크톱**: 피치 좌(2/3) + 우측 패널(슬라이더 상단, 예측 하단 1/3) `[설계 결정]`

- 첫 진입: 4-3-3 + 슬라이더 중앙값으로 **예측치가 이미 표시된 상태** (F8, P5)
- 점진적 공개: 세부 설정(포메이션 변경·정밀 모드·HOPs)은 접힌 상태로 시작 (P5)
- 드래그 피드백은 손가락 위 오프셋 표시 (P5)

### S2a — 리플레이 경기 선택

- 3경기 카드 세로 스택(모바일)/가로 3열(데스크톱): 상대 국기(flag-icons) + 라운드 + 개최지 + 스코어 (F9, P11 확정값)
- 카드 하단에 "이 경기, 당신이라면?" 톤의 진입 카피 `[설계 결정]` — 비하 금지 검수 대상

### S2b — 리플레이 재생·개입

```
┌─────────────────────┐
│ 경기 헤더 (스코어·개최지)│
├─────────────────────┤
│ 승률 라인차트 (F12)    │  ← 실제 vs 개입 시나리오 병렬
├─────────────────────┤
│ 이벤트 타임라인 (F10)  │  ← P11 확정 분 단위, 시점 선택 = 개입점
├─────────────────────┤
│ [이 시점에 개입하기]    │  → 전술보드(F2~F4 재사용) 오버레이
└─────────────────────┘
```

- 개입 후: 실제 결과 vs 재예측 확률 **나란히 비교** 패널 (F11). 단정 아닌 확률+밴드 표기
- xG 표기부에 "Opta xG" 라벨 상시 (P11)

### S3 — 공유 수신 랜딩

- URL 상태 복원 → 수신자 브라우저에서 자동 재추론 (F13, P8 — 서버 불필요)
- 구성: 복원된 전술 카드(보낸 사람의 전술) + 재추론 확률 + **"내 전술로 바꿔보기" CTA → S1 전환**(상태 이어받기)
- 이 화면이 바이럴 루프의 착지점 — 수신자가 1탭으로 조작 경험에 진입하는 것이 설계 목표 (P8 공통 퍼즐 비교)

### 화면 상태 변형 (전 화면 공통)

| 상태 | 처리 | 근거 |
|---|---|---|
| 초기 로딩 | 스켈레톤 + 기본 통계 우선 표시 — 모델 로딩을 기다리게 하지 않음 | P4·P5 |
| 추론 중 | 직전 결과 유지 + 갱신 인디케이터 (화면 비우지 않음) | P5 |
| 정밀 시뮬레이션 중 | 진행률 + 취소 (10초 임계) | P5·P12 |
| wasm 실패 | 순수 JS 통계 폴백 — 예측 패널 유지 | P4·P12 |
| **빈 상태** | **설계상 부재** — 모든 화면이 기본값으로 채워진 채 시작 | P5 |

## 사용 흐름 (U1~U4)

> 각 단계에 `화면 × 기능 × 지연 예산`을 교차 참조합니다.
> 골격 11절 요건("흐름의 모든 단계가 7·8절과 1:1 대응", 기획일관성 20)을 문서 구조로 강제합니다.

### U1 — 메인: 첫 방문 → 공유까지

| # | 단계 | 화면 | 기능 | 지연 예산 |
|---|---|---|---|---|
| 1 | 진입 — 기본 전술+예측이 이미 보임 | S1 | F8 | 초기 로딩 ≤3초 `[추정 설계목표]` |
| 2 | 선수 이동 (드래그 또는 클릭-클릭) | S1 | F2·F3 | 재배치 ~100ms |
| 3 | 슬라이더 조정 → 즉시 프리뷰 | S1 | F4 | ≤100ms |
| 4 | debounce 후 시뮬레이션 자동 실행 | S1 | F5·F6 | 빠른 모드 ≤1초 목표 |
| 5 | 확률 갱신 (배열+밴드, HOPs 옵션) | S1 | F7 | — |
| 6 | (선택) 정밀 모드 — 진행률+취소 | S1 | F6 | ≤10초, 초과 시 진행률 |
| 7 | 공유 — 카드 확인 → 공유 시트/클립보드 | S1 | F13~F16 | — |

### U2 — 리플레이: 선택 → 개입 → 비교

| # | 단계 | 화면 | 기능 |
|---|---|---|---|
| 1 | 모드 탭에서 리플레이 진입 | S1→S2a | — |
| 2 | 경기 카드 선택 (3경기, P11 확정값) | S2a | F9 |
| 3 | 타임라인·승률 라인차트 재생 | S2b | F10·F12 |
| 4 | 개입 시점 선택 → 전술보드 오버레이 | S2b | F11 (F2~F4 재사용) |
| 5 | 재예측 → 실제 결과와 나란히 비교 | S2b | F11·F5·F6 |
| 6 | 비교 결과 공유 (경기 식별자 포함 카드) | S2b | F13~F16 |

- 5단계 비교 카피는 "다른 선택지" 관점만 — 카피 단위 비하 금지 검수 (P8·비하 금지)

### U3 — 공유 수신: URL → 재추론 → 전환

| # | 단계 | 화면 | 기능 |
|---|---|---|---|
| 1 | 카톡 미리보기 카드(OG) 확인 — 앱 키 없이 성립 | (카톡) | F14 |
| 2 | 링크 진입 → 전술 상태 복원 | S3 | F13 |
| 3 | 수신자 브라우저에서 자동 재추론 | S3 | F5·F6 |
| 4 | "내 전술로 바꿔보기" → 전술보드로 전환 (상태 이어받기) | S3→S1 | — |
| 5 | 수정 후 재공유 → 바이럴 루프 | S1 | F13~F16 |

### U4 — 심사자 10분 시나리오 (모바일 전제)

> 각 단계가 심사 4항목 중 무엇을 보여주는지 주석. **설치·가입·키 입력 0회**를 동선으로 증명.

| # | 분 | 행동 | 심사 항목 노출 |
|---|---|---|---|
| 1 | 0~1 | 배포 URL 진입 — 즉시 완성 상태 화면 | 완성도 (빈 화면 없음) |
| 2 | 1~3 | 선수 드래그 + 슬라이더 → 확률이 실시간 반응 | 감동 경험 (조작 직관성) · 참신성 (온디바이스 예측) |
| 3 | 3~5 | 확률 시각화 탐색 (배열·밴드·HOPs) | 완성도 · 참신성 |
| 4 | 5~8 | 리플레이 진입 → 개입 → 실제와 비교 | 참신성 (리플레이 개입) · 감동 경험 |
| 5 | 8~9 | 공유 카드 생성 → 카톡 미리보기 확인 | 완성도 (키 없이 전 기능) |
| 6 | 9~10 | (심사자 재량) 키보드 조작·오프라인 재방문 확인 | 완성도 (접근성·내성) |

- 이 시나리오는 기획서 11절 원고와 시연 영상 콘티(전체 흐름: 시작 화면→배치·설정→인터랙션→결과)의 공통 원본이 된다 (대회요강 시연 영상 요건 정합)
