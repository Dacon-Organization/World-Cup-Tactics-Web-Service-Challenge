# 14. 기술 스택과 배포

> 분량 목표: 0.5p · 방어: 완성도 25 (키 없이 심사) · 근거: P4·P8

---

**핵심 메시지 — 심사 조건(설치 없음·가입 없음·키 없음·모바일)을 스택 선택으로 구조적으로
보장한다.**

| 선택 | 내용 | 방어하는 심사 조건 |
|---|---|---|
| Next.js 15 (App Router) | 서버 컴포넌트 우선 — 첫 화면을 서버가 완성해 전송 | 진입 즉시 완성 화면 (JS 실패 시에도 백지 없음) |
| TypeScript strict + Zod | 정적 데이터(JSON)의 타입·스키마 검증 | 오류 없이 안정 동작 (완성도 25) |
| Tailwind + shadcn/ui | 모바일 우선 반응형 | 심사자 모바일 열람 전제 |
| Zustand + React Hook Form | 전술 상태 관리·폼 검증 | 조작 일관성 |
| **ONNX Runtime Web (wasm)** | 온디바이스 추론 — Web Worker에서 실행 | **외부 API 0개 → 키 불필요·외부 장애 없음** (P4) |
| Vercel 배포 | 정적 자산(모델 포함) 직접 서빙 + 동적 OG 이미지 (P8) | 심사 기간 상시 접근 가능한 배포 URL |
| Service Worker 캐싱 | 모델·wasm 캐시 — 재방문·불안정 네트워크 대응 | 심사 중 재현성 (P4) |

배포 구조는 단순하다: **정적 페이지 + 정적 모델 파일 + 브라우저 내 계산.** 데이터베이스도
백엔드 API도 없으므로, 심사 중 "서버가 죽어서 안 열리는" 계열의 장애가 구조적으로
존재하지 않는다. 유일한 서버 코드는 공유 카드(OG 이미지)의 즉석 렌더뿐이며, 이것이
실패해도 본 기능은 영향받지 않는다(기본 카드 폴백, 15절).

`[조판: 배포 아키텍처 한 컷 — 브라우저 상자 안에 "전술 상태 → 추론 → 시뮬레이션 → 시각화"
전체 루프를 그리고, 외부에는 Vercel(정적 서빙·OG)만. "외부 API: 0" 뱃지 강조.
캡션 "모든 추론은 브라우저 안에서 끝난다 (P4)"]`
