**핵심 메시지 — 심사자의 10분: 첫 진입부터 공유까지 끊기지 않는 동선. 설치·가입·키 입력 0회.**

이 절의 모든 단계는 7절의 화면(S1~S3)과 8절의 인터랙션 명세에 1:1로 대응한다.

## 메인 플로우 — 첫 방문에서 공유까지

| # | 행동 | 화면 | 시스템의 응답 |
|---|---|---|---|
| 1 | 배포 URL 진입 | S1 | 기본 전술(4-3-3·슬라이더 중앙값)의 **예측이 이미 표시된 완성 화면** (P5) |
| 2 | 선수 드래그 (또는 클릭-클릭·키보드) | S1 | ~100ms 재배치 애니메이션 → 예측 갱신 |
| 3 | 슬라이더 조정 | S1 | 100ms 내 피치 시각 프리뷰 → 손을 떼면 시뮬레이션 자동 실행 |
| 4 | 확률 확인 | S1 | 아이콘 배열 + 퍼센트 + 신뢰구간 밴드 3중 표기 (P12) |
| 5 | (선택) 정밀 모드 | S1 | 25,000회 — 진행률·취소 제공, 완료 시 좁아진 밴드 |
| 6 | 공유 | S1 | 내 전술 카드 → OS 공유 시트/클립보드 (P8) |

## 리플레이 플로우 — "다른 선택지"의 시험

모드 탭 → 경기 카드 선택(S2a) → 타임라인·승률 라인차트 재생(S2b) → **개입 시점 선택** →
전술보드 오버레이(조작법은 메인과 동일 — 재학습 불요) → 재예측 → **실제 결과와 시나리오를
나란히 비교** → 경기 식별자가 포함된 카드로 공유. 비교 화면은 항상 확률+밴드로 말하고,
"이겼을 것"이라는 단정은 어디에도 없다.

## 공유 수신자 플로우 — 바이럴 루프의 닫힘

카톡 미리보기 카드(앱 키 없이 OG 스크래핑으로 성립, P8) → 링크 진입(S3) → URL에서 전술
상태 복원 → **수신자 브라우저에서 자동 재추론** (P8) → "내 전술로 바꿔보기" 1탭 → 상태를
이어받은 채 S1 진입 → 수정 → 재공유. 서버에 아무것도 저장되지 않는다 — URL이 곧 전술이다.

## 심사자 10분 시나리오 — 심사 조건의 동선 증명

| 분 | 행동 | 확인되는 것 |
|---|---|---|
| 0~1 | 모바일로 URL 진입 | 설치·가입 없이 즉시 완성 화면 (빈 화면 없음) |
| 1~3 | 드래그 + 슬라이더 조작 | 조작 직관성 · 100ms 반응 · 확률 실시간 갱신 |
| 3~5 | 확률 시각화 탐색 (배열·밴드·애니메이션) | 정직한 불확실성 표현 · 반복 횟수 명기 |
| 5~8 | 리플레이 진입 → 60' 개입 → 비교 | "내가 감독이라면"의 실제 작동 |
| 8~9 | 공유 카드 생성 → 카톡 미리보기 | **키 입력 0회로 전 기능 도달** |
| 9~10 | (재량) 키보드 조작·네트워크 탭 확인 | 접근성 경로 · 외부 API 호출 0건 |
