# 4. 서비스 개요 [대회 필수①]

> 분량 목표: 1p · 방어: **참신성 30** · 기획/구현 일관성 20 · 근거: P1

---

**핵심 메시지 — 전술을 만지면 승부가 바뀐다. 그리고 실제 3경기로 되돌아가 본다.**

「만약의 감독」은 세 개의 기능이 하나의 경험으로 이어지는 서비스다.
기능 나열이 아니라 **"감독의 하루"라는 단일 경험**으로 설계했다: 전술을 짜고(보드),
그 선택의 결과를 확인하고(예측), 실제 경기에 자신의 선택을 대입해 본다(리플레이).

## 3대 기능 축

| 축 | 무엇을 하는가 | 감독 경험에서의 역할 |
|---|---|---|
| ① **드래그앤드롭 전술보드** | 포메이션 프리셋 4종(4-3-3·4-4-2·3-4-3·3-5-2)에서 시작해 11명의 선수를 자유 배치. 슬라이더 4종(라인 높이·압박 강도·공격 폭·템포)으로 팀 성향 조정 | 감독의 **손** — 전술을 직접 만진다 |
| ② **온디바이스 ML 승부 예측** | 배치·슬라이더가 바뀔 때마다 직접 학습한 경량 모델이 **사용자의 브라우저 안에서** 승/무/패 확률을 재계산. 몬테카를로 시뮬레이션(5,000/25,000회)과 신뢰구간 병기 (P12) | 감독의 **판단 근거** — 선택의 결과를 본다 |
| ③ **한국 3경기 리플레이** | 실제 조별리그 3경기의 이벤트 타임라인을 재생하고, 임의 시점을 골라 "이 시점에 개입하기" — 전술을 바꿔 재예측하고 실제 결과와 나란히 비교 | 감독의 **복기** — 다른 선택지를 시험한다 |

여기에 부가 기능으로 **결과 공유**(내 전술 카드 + URL, 12절)가 바이럴 루프를 잇는다.

## 대회 취지와의 정합

대회가 예시로 든 "드래그앤드롭으로 선수를 배치하고 포메이션을 구성하는 전술 보드"와
"실제 경기 데이터를 기반으로 '내가 감독이었다면' 시나리오를 시뮬레이션하는 서비스"는
각각 이 서비스의 ①과 ③에 그대로 대응한다. 우리는 두 예시를 **하나의 서비스로 결합**하고,
그 사이를 ②(온디바이스 예측)로 연결했다 — 이 연결이 시장에 없는 조합이다.

## 포지션 한 줄 예고

조사한 36+개 전술보드·라인업 빌더·예측 서비스 중 "개별 선수 배치 변경 → 즉시 승부 예측"을
결합한 곳은 없었다 (P1). 상세한 경쟁 분석과 반박 방어는 5절에서 다룬다.

`[조판: 3기능 축 개념도 — 보드(손)→예측(판단)→리플레이(복기)를 순환 화살표로 잇고
중앙에 "감독 경험" 라벨. 부가 기능(공유)은 순환 밖 접선으로. 캡션 "세 기능이 하나의 경험을 이룬다"]`
