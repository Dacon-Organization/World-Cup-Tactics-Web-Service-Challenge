# 2. 한 장 요약

> 분량 목표: 1p · 방어: 전체 + **기획/구현 일관성 20** (약속 범위 고정)

---

## 무엇을 만드는가 — 한 문장

**전술을 만지면(드래그앤드롭) 승부가 바뀌고(온디바이스 ML 예측), 실제 3경기로 되돌아가
"다른 선택지"를 시험할 수 있는(리플레이) 웹서비스 — 「만약의 감독」.**

## 4블록 요약

| 문제 | 해법 |
|---|---|
| 한국은 조 3위 12팀 중 **10위** — 8위 진출선 바로 아래에서 대회를 마쳤다 (P6·P11). "다른 선택지가 있었다면?"이라는 질문은 모두의 것인데, 그것을 **직접 시험해 볼 도구가 없다** | 브라우저 안에서 완결되는 감독 경험: 선수 배치·전술 슬라이더를 조작하면 **심사자의 기기에서 직접 학습한 ML 모델이 즉시 승/무/패 확률을 재계산**한다 — 서버도, API 키도 없이 |

| 차별점 | 검증 계획 |
|---|---|
| 조사한 36+개 전술보드·예측 서비스 어디에도 "**개별 선수 배치 변경 → 즉시 승부 예측**" 결합이 없다 (P1). 여기에 실제 3경기 리플레이 개입과 공유 루프를 얹는다 | 성능은 공개 실측 정합 범위(RPS 0.19~0.22)를 방어선으로 삼고(P12), 몬테카를로 반복 횟수·신뢰구간을 화면에 항상 병기한다. 모든 수치는 부록(16절)에서 출처로 역추적된다 |

## 이 기획서가 심사 기준에 답하는 방식

| 심사 항목 | 배점 | 이 기획서의 답 (해당 절) |
|---|---|---|
| 참신성 | 30 | 시장 공백 증명(5) · 온디바이스 ML(10) · 리플레이 개입(4) · 공유 설계(12) |
| 감동 경험 설계 | 25 | 100ms 조작감·감독의 언어·빈 화면 없는 온보딩(6·7·8·11) |
| 완성도 | 25 | 외부 의존 0의 구조(10·14) · 지연 예산과 강등 계획(8·15) |
| 기획/구현 일관성 | 20 | **이 페이지의 약속 범위**(아래) ↔ 로드맵(15) 1:1 대조 |

## 약속 범위 — 이 페이지에 쓴 것만 약속한다

| 구분 | 기능 | 구현 우선순위 |
|---|---|---|
| **핵심 (필수 구현)** | ① 드래그앤드롭 전술보드 (포메이션 프리셋 4종 + 자유 배치 + 클릭-클릭·키보드 대안) | 1순위 |
| **핵심 (필수 구현)** | ② 온디바이스 ML 승부 예측 (슬라이더 4종 + 몬테카를로 시뮬레이션 + 확률 시각화) | 1순위 |
| **핵심 (필수 구현)** | ③ 한국 3경기 리플레이 (타임라인 + "이 시점에 개입하기" + 승률 라인차트) | 2순위 |
| **부가** | ④ 결과 공유 (URL 인코딩 + 동적 OG 카드 + Web Share/클립보드 폴백) | 3순위 |

일정 압박 시 ④→③ 순으로 축소하되 ①·②의 수용 기준은 타협하지 않는다(15절).
여기 없는 기능은 이후 어느 절에서도 약속하지 않는다.

`[조판: 4블록 요약을 2×2 카드 그리드로, 약속 범위 표는 핵심/부가 색 구분. 캡션 "약속 범위 — 15절 로드맵과 1:1 대응"]`
