# 5. 경쟁 분석과 차별화

> 분량 목표: 1p · 방어: **참신성 30** (직접 방어) · 근거: P1

---

**핵심 메시지 — 조사한 36+개 서비스 어디에도 "개별 선수 배치 → 즉시 승부 예측"이 없다.**

## 조사 범위와 결과

전술보드·라인업 빌더 지정 6개와 인접 영역(FPL 최적화 도구·베팅 시뮬레이터·HF Space 데모 등)
30여 개를 조사했다 (P1).

**지정 6개 — 전부 예측 기능 없음** (P1):

| 서비스 | 핵심 기능 | 예측 |
|---|---|---|
| buildlineup.com | 라인업 이미지 제작·공유 | 없음 |
| chosen11.com | 라인업 제작 | 없음 |
| lineup-builder.co.uk | 라인업 제작 (전술 옵션 다양) | 없음※ |
| footballuser.com | 전술 보드 | 없음 |
| tactical-board.com | 화이트보드형 전술판 | 없음 |
| sharemytactics.com | 라인업 공유 | 없음 |

※ lineup-builder.co.uk의 "38-0-0" 모드는 선수 능력치 단순 합산으로 시즌 성적을 추산하는
기능이 있어 엄밀히는 "완전 무예측"이 아니다 (P1). 다만 경기 단위 승부 예측도, 배치·전술과의
결합도 아니므로 아래 공백 판단을 바꾸지 않는다.

**인접 30여 개 — 예측은 있으나 조작 단위가 다름** (P1): 예측 기능을 가진 서비스들의 조작
단위는 "팀 선택" 또는 "팀 레이팅 슬라이더"에 그친다. **개별 선수 배치·포메이션 변경과 결합된
예측은 부재**했다 [추정 — P1 조사 범위 내].

## 포지셔닝 — 두 축의 교차점이 비어 있다

X축을 "조작 밀도"(팀 단위 선택 → 개별 선수 배치), Y축을 "예측 지능"(없음 → 통계 추산 →
ML 예측)으로 놓으면, 기존 서비스는 두 군집으로 나뉜다: 조작은 정밀하지만 예측이 없는
라인업 빌더군(우하단)과, 예측은 있지만 조작이 성긴 시뮬레이터군(좌상단).
**우상단 — 정밀 조작 × ML 예측 — 이 서비스의 자리다.**

`[조판: 경쟁 포지셔닝 맵 — X: 조작 밀도, Y: 예측 지능. 지정 6개는 우하단 군집,
FPL·베팅류는 좌상단 군집, 본 서비스는 우상단 단독 배치. 캡션 "빈 사분면 (P1)"]`

## 반박 방어 — "왜 아무도 안 했을까"

심사자가 물을 수 있다: *기술적으로 어려워서 비어 있는 것 아닌가?*
조사 결과의 해석은 다르다. 라인업 빌더류의 수익 모델은 "라인업 이미지의 소셜 공유"이며,
예측 기능은 이 모델에 필요하지 않았기 때문으로 추정된다 (P1 [추정]).
즉 이 공백은 기술 장벽이 아니라 **사업 모델의 선택**이 남긴 자리다. 실제로 본 기획은
경량 모델 + 브라우저 추론이라는 검증된 부품(10절)만으로 이 결합을 구현한다.

같은 조사의 다른 함의도 기록해 둔다: 선례가 없다는 것은 **참신성의 증거인 동시에 벤치마크
부재라는 완성도 리스크**다 (P1). 이 리스크는 15절의 강등 계획(시뮬레이션 모드 조정·통계
폴백)으로 관리한다.
